## 4. DRAG AND DROP — CURSOR BEHAVIOR OVER TEXT

Update the cursor behavior so that drag-and-drop does not conflict with text selection and copying.

**Current behavior (problematic):**
- When hovering anywhere on a row, the `grab` cursor appears, making it difficult to select and copy text values.

**Updated behavior:**

**Over text content:**
- When the cursor hovers over any text content within a cell (Material Code, Item Description, FPR values, Planned Order numbers, Production Order numbers, FPR Notes, Special Remarks, Volume values, date values, or any other displayed text), the cursor changes to `text` (I-beam / typeface cursor).
- The user can click and drag to select text for copying. This takes priority over drag-and-drop initiation.
- Text selection works normally: click-drag to select, double-click to select a word, triple-click to select a line. Right-click context menu for Copy is available.

**Over non-text areas (empty cell space, row background, padding):**
- When the cursor hovers over empty/non-text areas of a row (cell padding, whitespace, row background between columns), the `grab` cursor appears, indicating the row can be dragged.
- Click-and-hold in these areas initiates the drag-and-drop operation.

**Over interactive elements:**
- Dropdowns, input fields, buttons, lock icons, calendar pickers: Standard cursors for their respective element types (`pointer` for buttons/icons, `text` for input fields, etc.). These do NOT initiate drag-and-drop (as previously defined).

**Over the drag handle icon (grip dots on the left):**
- The drag handle icon always shows the `grab` cursor.
- Click-and-hold on the drag handle always initiates drag-and-drop regardless of nearby text.
- This serves as the guaranteed drag initiation point for users who want to drag from a specific spot.

**Implementation approach:**
- Text elements within cells should have `cursor: text` and should stop event propagation for mousedown/dragstart to prevent the row-level drag handler from activating.
- Non-text areas and the drag handle retain the row-level drag behavior with `cursor: grab` (and `cursor: grabbing` while dragging).
- Ensure that after text selection is complete (mouseup), the drag functionality is not accidentally triggered.

**Summary of cursor mapping:**
| Area                          | Cursor    | Click behavior              |
|-------------------------------|-----------|-----------------------------|
| Text content in cells         | `text`    | Text selection / copy       |
| Empty cell space / padding    | `grab`    | Drag-and-drop initiation    |
| Drag handle icon (left side)  | `grab`    | Drag-and-drop initiation    |
| Buttons / lock icons          | `pointer` | Button action               |
| Input fields                  | `text`    | Focus / edit                |
| Dropdowns                     | `pointer` | Open dropdown               |
| Calendar / time pickers       | `pointer` | Open picker                 |
