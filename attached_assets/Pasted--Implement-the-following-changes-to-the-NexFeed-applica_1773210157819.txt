## 3. SAP UPLOAD — "CANCEL PO" STATUS FROM PO STATUS COLUMN

When uploading a SAP Excel file, the system must read the "PO Status" column and handle cancelled orders.

**Column mapping:**
- SAP column: "PO Status" (case-insensitive matching: "po status", "PO Status", "PO STATUS", etc.)
- If the value is "Cancelled" (case-insensitive: "cancelled", "CANCELLED", "Cancelled"):
  - Set the order's status in the app to "Cancel PO"

**"Cancel PO" behavior:**
- "Cancel PO" is a DISTINCT status from user-initiated "Cancelled".
  - "Cancelled" = user manually cancelled the order in the app
  - "Cancel PO" = SAP planned order was cancelled at source

**FPR Notes auto-entry for Cancel PO:**
- When a Cancel PO order is uploaded, automatically set FPR Notes to:
  ```
  Cancelled: Cancelled from SAP Planned Order — [Upload Date, HH:MM AM/PM]
  ```
- Example:
  ```
  Cancelled: Cancelled from SAP Planned Order — March 10, 2026 08:00 AM
  ```

**Row behavior for Cancel PO (same as user-cancelled):**
- Row moves to the BOTTOM of the interactive table, below all active orders.
- Greyed out styling:
  - All text: light grey (#c0c7d0)
  - Background: light grey tint (#f5f5f5)
  - Opacity: 0.7
  - Strikethrough on Item Description
- Status badge shows "Cancel PO" in red (#e53935) with a distinct label so users can differentiate from user-cancelled.
- Excluded from scheduling cascade.
- NOT draggable.
- Auto-removed at midnight of the next day and moved to Order History > Cancelled Orders.

**Status badge distinction:**

| Status | Badge Color | Badge Text | Source |
|---|---|---|---|
| Cancelled | Red (#e53935) | Cancelled | User manually cancelled in app |
| Cancel PO | Red (#e53935) with orange outline (#fd5108) | Cancel PO | SAP PO Status = Cancelled |

The slight visual difference (orange outline on Cancel PO) helps users distinguish between the two at a glance.

**Cancel PO orders are read-only:**
- Users cannot edit any fields on a Cancel PO order.
- Users cannot change the status of a Cancel PO order (it is locked).
- If the user needs to reinstate a Cancel PO order, they must do so in SAP and re-upload.
