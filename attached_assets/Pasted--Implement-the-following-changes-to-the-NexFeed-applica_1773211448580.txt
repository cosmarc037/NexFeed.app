## 1. CANCEL ORDER — CONFIRMATION DIALOG WITH REASON

When a user changes an order's status to "Cancel", a confirmation dialog must appear requiring a cancellation reason before the action is processed.

**Dialog trigger:**
- User selects "Cancel" from the status dropdown on any order row.
- The status does NOT change immediately — the dialog opens first.
- If the user dismisses the dialog, the status reverts to its previous value.

**Dialog layout:**

```
╔══════════════════════════════════════════════════╗
║  ⚠ Cancel Order                                  ║
║                                                   ║
║  Are you sure you want to cancel this order?      ║
║                                                   ║
║  FPR: [260309]                                    ║
║  Item: [Elite XP Growex 2, Pellet 50kg]           ║
║  Volume: [80 MT]                                  ║
║                                                   ║
║  Cancellation Reason *                            ║
║  ┌─────────────────────────────────┐              ║
║  │ Select a reason          ▼     │              ║
║  └─────────────────────────────────┘              ║
║                                                   ║
║  Additional Notes (optional)                      ║
║  ┌─────────────────────────────────┐              ║
║  │                                 │              ║
║  │                                 │              ║
║  └─────────────────────────────────┘              ║
║                                                   ║
║         [Go Back]    [Confirm Cancel]             ║
╚══════════════════════════════════════════════════╝
```

**Cancellation reason dropdown options:**
- Client request
- Material unavailable
- Scheduling conflict
- Duplicate order
- Quality issue
- Management decision
- Other

**Validation:**
- Cancellation Reason dropdown is REQUIRED (*). The "Confirm Cancel" button is disabled until a reason is selected.
- Additional Notes is optional free-text.
- If "Other" is selected, the Additional Notes field becomes REQUIRED (user must explain).

**Button styles:**
- [Go Back]: Grey outlined button. Closes dialog, reverts status.
- [Confirm Cancel]: Red filled button (#e53935), white text. Processes the cancellation.

## 2. CANCEL ORDER — FPR NOTES + ROW BEHAVIOR

**FPR Notes auto-entry on cancel:**
- When the user confirms cancellation, the FPR Notes field is automatically updated with:
  ```
  Cancelled: [Reason] — [Month DD, YYYY HH:MM AM/PM]
  ```
- If Additional Notes were provided:
  ```
  Cancelled: [Reason] — [Month DD, YYYY HH:MM AM/PM]. [Additional Notes]
  ```
- Examples:
  ```
  Cancelled: Material unavailable — March 10, 2026 02:30 PM
  ```
  ```
  Cancelled: Other — March 10, 2026 02:30 PM. Supplier delayed shipment by 2 weeks.
  ```
- This text is APPENDED to any existing FPR Notes content (does not overwrite).

**Row behavior after cancellation in the interactive table:**

**Immediate visual changes:**
- The cancelled order row moves to the BOTTOM of the interactive table, below all active orders.
- The row is greyed out:
  - All text: light grey (#c0c7d0)
  - Background: light grey tint (#f5f5f5)
  - Opacity: 0.7
  - NO strikethrough — text remains normal style, just greyed out
- Status badge shows "Cancel PO" in red (#e53935).

**Scheduling impact:**
- The cancelled order is EXCLUDED from the cascade.
- Downstream orders skip over the cancelled order as if it doesn't exist.
- When an order is cancelled, all downstream Completion Dates recalculate (they shift earlier since the cancelled order's production time and changeover are removed from the chain).

**Row ordering in the interactive table:**
```
Done orders (greyed out, at the top)
────────────────────────────────────
Active orders (Prio 1, 2, 3, ...) ← normal scheduling
────────────────────────────────────
Cancelled orders (greyed out, at the bottom)
```

**Cancelled orders are NOT draggable** — they cannot be reordered.

**Auto-removal after end of day:**
- At 12:00 AM (midnight) of the NEXT day after cancellation, the cancelled order is automatically removed from the interactive table.
- The order is moved to the Order History page under the "Cancelled Orders" tab.
- Example: Order cancelled on March 10 at 2:30 PM → removed from interactive table at March 11, 12:00 AM → appears in Order History > Cancelled Orders.

## 3. SAP UPLOAD — "CANCEL PO" STATUS FROM PO STATUS COLUMN

When uploading a SAP Excel file, the system must read the "PO Status" column and handle cancelled orders.

**Column mapping:**
- SAP column: "PO Status" (case-insensitive matching: "po status", "PO Status", "PO STATUS", etc.)
- If the value is "Cancelled" (case-insensitive: "cancelled", "CANCELLED", "Cancelled"):
  - Set the order's status in the app to "Cancel PO"

**"Cancel PO" is the single status for ALL cancelled orders:**
- User-cancelled orders → status = "Cancel PO"
- SAP-cancelled orders → status = "Cancel PO"
- Both use the same "Cancel PO" status label.

**FPR Notes auto-entry for SAP Cancel PO:**
- When a Cancel PO order is detected during upload, automatically set FPR Notes to:
  ```
  Cancelled: Cancelled from SAP Planned Order — [Upload Date, HH:MM AM/PM]
  ```
- Example:
  ```
  Cancelled: Cancelled from SAP Planned Order — March 10, 2026 08:00 AM
  ```

**Row behavior for SAP Cancel PO (same as user-cancelled):**
- Row moves to the BOTTOM of the interactive table, below all active orders.
- Greyed out styling:
  - All text: light grey (#c0c7d0)
  - Background: light grey tint (#f5f5f5)
  - Opacity: 0.7
  - NO strikethrough
- Status badge shows "Cancel PO" in red (#e53935).
- Excluded from scheduling cascade.
- NOT draggable.
- Auto-removed at midnight of the next day and moved to Order History > Cancelled Orders.

**Cancel PO orders from SAP are read-only:**
- Users cannot edit any fields on a SAP Cancel PO order.
- Users cannot change the status of a SAP Cancel PO order (it is locked).
- If the user needs to reinstate a Cancel PO order, they must do so in SAP and re-upload.
