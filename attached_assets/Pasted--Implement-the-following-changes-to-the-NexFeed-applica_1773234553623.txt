### WHAT HAPPENS ON CONFIRM MERGE:

1. **Portion 1 absorbs Portion 2:**
   - Portion 1's volume updates to: Portion 1 + Portion 2 (original total).
   - Production hours, batches, bags recalculate based on merged volume.

2. **Portion 2 is removed:**
   - Portion 2 row is deleted from the interactive table.
   - Prio numbers re-sequence.

3. **Tags removed:**
   - "Cut" tag is removed from the merged order.

4. **Status reset:**
   - Merged order's status resets to "Plotted".

5. **Position:**
   - Merged order stays at Portion 1's current position.

6. **FPR Notes:**
   - Append to existing notes:
     ```
     Merged: Portions merged back from 16 MT + 8 MT to 24 MT — March 11, 2026 10:30 AM
     ```
   - Note text in dark text (not colored — this is a neutral action).

7. **Cascade recalculation:**
   - Downstream orders recalculate completion dates.

### MERGE BACK — WHICH ORDER TRIGGERS IT:

- The user can select "Merge Back" from EITHER Portion 1 or Portion 2.
- Both trigger the same dialog and same merge result.
- The merged order always takes Portion 1's position (the earlier Prio).

### EDGE CASES FOR MERGE BACK:

**Edge case 1 — One portion is "Done", the other is active:**
- "Merge Back" is NOT available if either portion has status "Done".
- Hide "Merge Back" from dropdown if the OTHER portion's status is "Done".
- Tooltip: "Cannot merge — the other portion is already completed."

**Edge case 2 — One portion is cancelled:**
- "Merge Back" is NOT available if either portion is cancelled.
- Hide "Merge Back" from dropdown.
- Tooltip: "Cannot merge — the other portion is cancelled."

**Edge case 3 — One portion is part of a combined group:**
- "Merge Back" is NOT available if either portion is part of a combined group.
- The user must uncombine first, then merge back.
