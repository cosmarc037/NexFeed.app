## FIX: AI STOCK INSIGHTS — MARKDOWN BOLD NOT RENDERING

The AI Stock Insights section is displaying raw markdown bold markers (**text**) as literal asterisks instead of rendering them as actual bold text.

**Current (broken):**
```
**A) Overview** In the next 10 days, there are 20 products...
**B) Urgent — Production Needed Soon** - **Gallimax 3 Maintenance...
```

**Expected (correct):**
```
A) Overview In the next 10 days, there are 20 products...
B) Urgent — Production Needed Soon - Gallimax 3 Maintenance...
```
Where "A) Overview", "B) Urgent — Production Needed Soon", and product names are rendered in BOLD, not with visible asterisks.

---

### THE FIX

The AI response text contains markdown-style bold markers (`**text**`). The rendering component must parse and convert these to actual bold HTML/JSX before displaying.

**Option A — Simple regex replacement (recommended):**

```javascript
function renderAIText(text) {
  // Replace **bold text** with <strong>bold text</strong>
  const parts = [];
  const regex = /\*\*(.*?)\*\*/g;
  let lastIndex = 0;
  let match;
  
  while ((match = regex.exec(text)) !== null) {
    // Add text before the match
    if (match.index > lastIndex) {
      parts.push(text.substring(lastIndex, match.index));
    }
    // Add the bold text
    parts.push(<strong key={match.index}>{match[1]}</strong>);
    lastIndex = regex.lastIndex;
  }
  
  // Add remaining text after last match
  if (lastIndex < text.length) {
    parts.push(text.substring(lastIndex));
  }
  
  return parts;
}
```

**Option C — dangerouslySetInnerHTML with sanitization:**

```javascript
function renderAIText(text) {
  // Replace **bold** with <strong>bold</strong>
  const html = text.replace(/\*\*(.*?)\*\*/g, '<strong>$1</strong>');
  
  // Also handle line breaks
  const withBreaks = html.replace(/\n\n/g, '<br/><br/>');
  
  return <span dangerouslySetInnerHTML={{ __html: withBreaks }} />;
}
