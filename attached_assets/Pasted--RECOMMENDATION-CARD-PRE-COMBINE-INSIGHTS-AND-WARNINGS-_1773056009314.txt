### RECOMMENDATION CARD — PRE-COMBINE INSIGHTS AND WARNINGS

Add a pre-combine insights and warnings section within each recommendation card, positioned between the Combined Volume total and the action buttons. This section proactively shows the user potential issues BEFORE they click Approve.

**Card layout (updated, top to bottom):**
1. Group header: "Combine Group [#]"
2. Shared details: Material Code, Item Description, Form, Line
3. Active orders list (each with X button on the right)
4. Combined Volume total (recalculates based on active orders only)
5. Divider
6. Excluded section (removed orders with [+ Re-add] button) — hidden if no orders are removed
7. **Pre-Combine Insights & Warnings section (NEW)**
8. Action buttons: [Approve] [Dismiss]

---

**Pre-Combine Insights & Warnings Section:**

This section is generated by the AI and recalculates in real-time as the user adds/removes orders from the group via the X and Re-add buttons.

**Section label:** "Insights & Warnings" (small, bold, Dark Grey #2e343a text)

**Content types:**

**Warnings (displayed with ⚠ orange icon):**

These are potential issues that may block or complicate the combine:

• **Scheduling conflict:**
  "⚠ Combining these orders will push downstream order FPR: [value] ([Item Description]) past its Avail Date by [X] hours/days."
  "New Completion Date: [date] → Avail Date: [date]"
  (List each affected downstream order separately)

• **Line capacity exceeded:**
  "⚠ Combined volume of [X] MT would exceed remaining capacity on Line [X] ([current] MT / 1,000 MT remaining: [Y] MT)."

• **Order already in a combine group:**
  "⚠ FPR: [value] is already part of another combined group. It must be un-combined first."

• **Incompatible status:**
  "⚠ FPR: [value] has status '[status]' which is not eligible for combining. Only active orders (Plotted, Hold, Cut) can be combined."

• **Batch size multiple warning:**
  "⚠ Combined volume of [X] MT is not a multiple of the Batch Size ([Y]). The system will round up to [Z] MT."

**Informational insights (displayed with ℹ blue icon):**

These are helpful context and recommendations, not blockers:

• **Production hours estimate:**
  "ℹ Combined production will take approximately [X.XX] hours ([X] batches at [Run Rate] MT/hr)."

• **Completion date estimate:**
  "ℹ Estimated Completion Date: [MMMM DD, YYYY HH:MM AM/PM]"

• **Avail date comparison:**
  "ℹ Earliest Avail Date among these orders: [date]. Estimated completion is [X hours/days] before/after this deadline."

• **Volume context:**
  "ℹ Combined volume meets the minimum threshold of [X] MT for [Feedmill Name]."
  OR
  "ℹ Combined volume of [X] MT is below the recommended minimum of [Y] MT for [Feedmill Name]."

• **No run rate available:**
  "ℹ Form is 'M' (Mash) — no run rate available. Production Hours cannot be estimated."

**Blocking behavior:**

When ANY warning indicates a blocking issue (scheduling conflict or line capacity exceeded), the [Approve] button is:
- Disabled (greyed out)
- Tooltip on hover: "Resolve warnings before combining"

The specific blocking warnings are:
- Scheduling conflict (downstream orders would miss Avail Date)
- Line capacity exceeded

Non-blocking warnings (already combined, incompatible status, batch size multiple) still allow the user to click Approve but will show the appropriate error in the confirmation dialog if the issue persists.

When there are NO blocking warnings, the [Approve] button remains enabled.

**Design:**

- Background: Light yellow (#fff9c4) for warnings, Light blue (#e8f4fd) for informational insights.
- Each item is a compact card/row within the section.
- Warning items are listed FIRST (above informational insights).
- If there are no warnings and no insights (rare edge case), the section displays: "✓ No issues detected. Ready to combine."
- The section has a maximum height with a scrollbar if there are many items, to avoid the card becoming too tall.
- A small refresh icon next to the section label to regenerate insights on demand.

**Real-time recalculation:**

- When the user removes an order via the X button, the insights and warnings recalculate immediately.
  Example: If removing an order resolves a scheduling conflict, the warning disappears and the [Approve] button becomes enabled.
- When the user re-adds an order via [+ Re-add], the insights recalculate again.
- The Combined Volume, Production Hours estimate, Completion Date estimate, and downstream conflict check all update in real-time.
