2. The AI service module should expose functions for each AI use case:
   - `suggestStartDateTime(orderData)` — Suggests Start Date and Start Time by working backwards from Avail Date as the deadline. Considers Suggested Volume, Number of Batches, Batch Size, Production Hours, Changeover Time, Run Rate, and line capacity. Guidelines: Small orders (<50 MT) start 1–2 days before, Medium (50–200 MT) 2–3 days, Large (>200 MT) 3–5 days. Default Start Time: 08:00 AM unless AI determines otherwise. Returns blank if Avail Date is non-date text.
   - `generateSmartRecommendations(ordersInView)` — Generates contextual recommendations for the current tab/sub-tab view. Identifies combine/cut opportunities (same Material Code), orders below minimum MT threshold, stagnant orders, and cross-line opportunities (for "All" tab). Returns empty state message if no orders exist.
   - `generateSmartAlerts(allOrders)` — Generates natural language alert summaries covering: URGENT (Avail Date within 2 business days, not Done), COMPLETION APPROACHING (Completion Date within 2 business days), COMPLETION OVERDUE (Completion Date passed, not Done), OVERDUE/STAGNANT (Avail Date passed, not Done), INCOMPLETE (red X readiness), LINE CAPACITY (line at full capacity).
   - `generateOverviewSummary(allOrders, lineCapacities)` — Generates a natural language summary for the Overview tab covering production health, line capacity, key action items, and priority recommendations.
   - `generateAnalyticsInsights(allOrders)` — Generates contextual insights about order trends, demand patterns, high-demand products, slow-moving items, and production optimization recommendations.
   - `chatWithAssistant(messageHistory, currentAppState)` — Handles Smart Assistant (chatbot) conversations. The assistant can answer questions about current order status, provide scheduling recommendations, explain urgent/flagged orders, summarize production state and line capacity, and assist with app feature queries.

3. For each function, construct a system prompt that provides the AI with relevant context about the orders and the specific task. Send the order data as part of the user message. Parse the AI's response and return structured data that the frontend can render.
