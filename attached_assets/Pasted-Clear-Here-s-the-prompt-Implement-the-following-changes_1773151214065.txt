### INSIGHT 1 — COMBINED PRODUCTION CALCULATION

When orders are grouped for combining, the AI must calculate and display the combined production metrics.

**Calculations:**
- **Combined Total Volume (MT):** Sum of all orders' Volume in the group.
- **Batch Size:** Same across all orders in the group (since Batch Size is a combine matching criteria).
- **Number of Batches:** Calculated from Combined Total Volume ÷ Batch Size (rounded up to nearest whole number).
- **Total Bags:** Calculated based on the combined volume and packaging (same formula used elsewhere in the app).
- **Total Production Hours:** Number of Batches × Run Rate per batch.

**Display in the Insights section of the recommendation card:**

```
📊 Combined Production Summary
───────────────────────────────────────────────
Combined Volume:        12 MT
Batch Size:             4 MT
Number of Batches:      3
Total Bags:             240
Production Time:        4 hours 30 min
───────────────────────────────────────────────
```

**Important:** The Number of Batches is NOT simply the sum of individual orders' batches. It is recalculated from the combined volume and batch size. Example:
- Order A: 8 MT, Batch Size 4 MT = 2 batches
- Order B: 4 MT, Batch Size 4 MT = 1 batch
- Combined: 12 MT ÷ 4 MT = 3 batches (not 2 + 1 = 3, but calculated fresh from 12 ÷ 4)
- In this case the result is the same, but the system must always recalculate from combined volume, not sum individual batch counts, to handle rounding correctly.

### INSIGHT 3 — SMART DISMISSAL RECOMMENDATION (GAP UTILIZATION)

The AI must proactively recommend NOT combining orders when keeping them separate allows non-dated orders (safety stocks, prio replenish) to be inserted in the time gap, maximizing plant utilization.

**Trigger condition:**
- The AI checks if there are ANY pending non-dated orders (prio replenish, safety stocks) in the same line that are currently unscheduled or placed at the end of the timeline.
- If non-dated orders exist AND keeping the orders separate would create a time gap between them where non-dated orders could be inserted, the AI recommends dismissing the combine.

**Logic:**
1. Calculate the time gap if orders are kept SEPARATE:
   - Order A completes at [time]. Order B starts at [time based on its own Avail Date anchor].
   - Gap = Order B Start - Order A Completion.
2. Check if any non-dated orders' production time fits within this gap.
3. If YES — recommend not combining.

**Display in the Insights section:**

```
💡 Recommendation: Consider NOT combining
───────────────────────────────────────────────
FPR 260311's Avail Date (March 28) is 6 days
after FPR 260309's Avail Date (March 22).

If kept separate:
• FPR 260309 completes: March 21, 02:15 PM
• FPR 260311 needs to start by: March 27, 09:45 PM
• Available gap: ~5 days 7 hours

Pending non-dated orders that could fill this gap:
• [Item Description] (safety stocks) — 4 hrs 30 min
• [Item Description] (prio replenish) — 3 hrs 15 min
• [Item Description] (safety stocks) — 6 hrs 00 min
  Total: 13 hrs 45 min

Keeping orders separate allows you to insert
these 3 non-dated orders in the gap, maximizing
plant utilization. Combining would eliminate
this opportunity.

[Dismiss — Keep Separate]  [Approve Anyway]
───────────────────────────────────────────────
```

**When this recommendation does NOT apply:**
- If there are NO pending non-dated orders in the line — the AI does not show this recommendation (no gap fillers available, so combining is fine).
- If the gap between orders is too small to fit any non-dated order — the AI does not show this recommendation.
- If both orders have the SAME Avail Date or very close Avail Dates (no meaningful gap) — the AI does not show this recommendation.

**Button behavior when recommendation is shown:**
- **[Dismiss — Keep Separate]:** Dismisses the combine recommendation. The orders remain separate. Same as the existing Dismiss button but with clearer labeling.
- **[Approve Anyway]:** The user overrides the AI recommendation and proceeds with the combine. The combine happens as normal. The History column logs: "Combined despite AI recommendation to keep separate — [Timestamp] by [User]."
