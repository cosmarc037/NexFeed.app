## 1. CUT ORDER — STATUS TRIGGERS DIALOG

When a user selects "Cut" from the status dropdown on any standalone order, a dialog appears. The status does NOT change immediately — the dialog opens first.

**Trigger:**
- User selects "Cut" from the status dropdown.
- Dialog opens.
- If user confirms, the order is split into two portions.
- If user cancels/goes back, the status reverts to its previous value.

**"Cut" is NOT available for:**
- Combined orders (lead or child) — hidden from combined order dropdown.
- Orders already tagged as "Cut" — hidden from dropdown (cannot cut twice).

**Example interaction:**

### DIALOG — BATCH SIZE VALIDATION

Both Portion 1 AND Portion 2 must be divisible by the batch size.

**Validation rules:**
- Portion 1 ÷ Batch Size must result in a whole number (no decimals).
- Portion 2 ÷ Batch Size must result in a whole number (no decimals).
- If either fails, show a validation error.

**Validation error display:**
- Error text appears below the Portion 1 input field.
- Red text (#e53935).
- Error messages:

| Scenario | Error Message |
|---|---|
| Portion 1 not divisible | "⚠ [X] MT is not divisible by batch size [Y]. Try [nearest lower] or [nearest higher] MT." |
| Portion 2 not divisible | "⚠ Remaining [X] MT is not divisible by batch size [Y]. Try [adjusted value] MT for Portion 1." |
| Portion 1 is 0 or empty | "⚠ Please enter a volume greater than 0." |
| Portion 1 equals total | "⚠ Portion 1 cannot equal the total volume. Leave some for Portion 2." |
| Portion 1 exceeds total | "⚠ Portion 1 cannot exceed the total volume of [X] MT." |

**Helpful suggestions in error messages:**

When the value is not divisible, calculate and suggest the nearest valid values:

```
Volume: 80 MT, Batch Size: 4.00
User enters: 30 MT

Error: "⚠ 30 MT is not divisible by batch size 4.00. Try 28 MT or 32 MT."

  28 ÷ 4 = 7 batches → Portion 2: 52 MT (52 ÷ 4 = 13 batches) ✅
  32 ÷ 4 = 8 batches → Portion 2: 48 MT (48 ÷ 4 = 12 batches) ✅
```

**"Confirm Cut" button is disabled** until both Portion 1 and Portion 2 pass validation.

## 3. CUT ORDER — WHAT HAPPENS ON CONFIRM

When the user confirms the cut:

**Step 1: Original order becomes Portion 1**
- The original order's volume is updated to Portion 1 value.
- The original order's production hours, batches, bags are recalculated based on new volume.
- Status is set to "Cut".
- A "Cut" tag/badge is added to the order.

**Step 2: New order created for Portion 2**
- A NEW order row is created with:
  - Same FPR, FG, SFG, Material Code, Category, Form, Batch Size, Line as the original.
  - Volume = Portion 2 value.
  - Production hours, batches, bags calculated based on Portion 2 volume.
  - Status = "Cut".
  - A "Cut" tag/badge is added.
  - Availability = same as original order.
  - Start Date/Time = blank (user must set).
  - Completion Date = blank initially (will be calculated by cascade if applicable).

**Step 3: Placement**
- Portion 1 stays in the original order's position.
- Portion 2 is placed IMMEDIATELY BELOW Portion 1.
- Prio numbers are re-sequenced.

```
Before cut:
Prio 5: FPR 260309, 80 MT

After cut (Portion 1 = 32 MT, Portion 2 = 48 MT):
Prio 5: FPR 260309, 32 MT [Cut]    ← original order, updated volume
Prio 6: FPR 260309, 48 MT [Cut]    ← new order, Portion 2
```

**Step 4: Both portions are independently movable**
- Unlike combined orders, cut portions do NOT need to stick together.
- The user can freely drag either portion to any position in the table.
- They are independent orders that happen to share the same FPR.

**Step 5: Cascade recalculation**
- Completion Dates recalculate for both portions and all downstream orders.

**Step 6: FPR Notes**
- Portion 1 FPR Notes appended with:
  ```
  Cut: Split from 80 MT to 32 MT (Portion 1 of 2) — March 11, 2026 10:30 AM
  ```
- Portion 2 FPR Notes set to:
  ```
  Cut: Split from 80 MT to 48 MT (Portion 2 of 2) — March 11, 2026 10:30 AM
  ```
- Note text displayed in purple (#7c3aed) to match the Cut status color.

## 6. CUT ORDERS — RECOMBINE

Cut portions CAN be recombined back together.

**How it works:**
- If both cut portions (Portion 1 and Portion 2) are selected for combining via Smart Combine or manual combine:
  - The two portions merge back into a single order.
  - Volume = Portion 1 + Portion 2 (back to original).
  - The "Cut" badge is removed from the merged order.
  - Production hours, batches, bags are recalculated.
  - FPR Notes appended with:
    ```
    Recombined: Merged back from 32 MT + 48 MT to 80 MT — March 12, 2026 02:00 PM
    ```

**Recombine is only possible when:**
- Both portions share the same original FPR.
- Both portions are on the same line.
- Neither portion is currently part of another combined group.

## 8. CUT ORDERS IN ORDER HISTORY

When cut orders are moved to Order History (via Done or Cancel):

- The "Cut" badge is retained in Order History.
- "Original: XX MT" reference is retained.
- Both portions may appear independently in Order History (since they are independent orders).
- They are NOT required to be adjacent in Order History.

## 9. EDGE CASES

**Edge case 1 — Cut order is cancelled:**
- The cut order follows normal cancellation flow (dialog, FPR notes, moves to bottom).
- The OTHER cut portion is NOT affected — it remains active and independent.
- The "Cut" badge remains on both portions.

**Edge case 2 — One cut portion is Done, the other is still active:**
- This is valid. Each portion operates independently.
- Portion 1 can be "Done" and at the top while Portion 2 is still active in the schedule.

**Edge case 3 — Cut order combined with a DIFFERENT order (not its other portion):**
- This is allowed. A cut portion can be combined with any compatible order via Smart Combine.
- The "Cut" badge remains alongside the "Lead" or "Child" badge.
- Example: Prio 5 shows both [Cut] and [Child] badges.

**Edge case 4 — User tries to cut a combined order:**
- "Cut" is hidden from combined order dropdown — not allowed.
- User must uncombine first, then cut.
