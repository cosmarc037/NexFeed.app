## 1. VOLUME (MT) — EDITABLE VIA OVERRIDE MECHANISM

Make the Volume (MT) column editable via an override mechanism:

**Override Mechanism:**
- By default, both Suggested Volume and Original Volume are read-only (locked).
- A small lock icon is displayed next to the Suggested Volume value.
- When the user clicks the lock icon, a confirmation dialog appears:
  "You are about to override the Suggested Volume. This will affect Number of Batches, Number of Bags, Production Hours, and Completion Date. Type 'Confirm' to proceed."
  Input field: User must type "Confirm" (case-insensitive).
  Buttons: [Proceed] (disabled until "Confirm" is typed) [Cancel]
- Upon confirmation, the field unlocks and the user can enter a custom Suggested Volume value.
- The lock icon changes to an "unlocked" state to indicate the value has been overridden.

**After Override:**
- All downstream fields recalculate based on the new Suggested Volume:
  - Number of Batches = new Suggested Volume / Batch Size
  - Number of Bags = new Suggested Volume / 50 * 1000
  - Production Hours = new Suggested Volume / Run Rate (if Run Rate exists)
  - Completion Date recalculates, then cascades to all downstream orders in the same feedmill line.

**Batch Size multiple warning:**
- If the user-entered value is NOT a multiple of the Batch Size, display a small warning icon (⚠) next to the Suggested Volume with a tooltip: "This volume is not a multiple of the Batch Size ([X])."
- The warning is informational only — it does NOT block the override.

**Display after override (3 lines):**
- Line 1: [User-entered Volume] MT (bold/prominent, with ⚠ if not a multiple of Batch Size)
- Line 2: "Original: [X] MT | System Suggested: [Y] MT" (smaller, grey text #a1a8b3)

**Display when NOT overridden (2 lines — current behavior):**
- Line 1: [Suggested Volume] MT (bold/prominent)
- Line 2: "Original: [X] MT" (smaller, grey text)

**Editability by status:**
- Plotted, Cut, Combine, Hold: Override mechanism available
- In production, On-going *, Done, Cancel PO: Read-only (no override)

**Volume values display as whole numbers — no decimal places.**

## 2. COMPLETION DATE — EDITABLE VIA OVERRIDE MECHANISM

Make the Completion Date column editable via an override mechanism:

**Override Mechanism:**
- By default, Completion Date is read-only (locked, calculated output).
- A small lock icon is displayed next to the Completion Date value.
- When the user clicks the lock icon, a confirmation dialog appears:
  Title: "Override Completion Date"
  Message: "You are about to override the calculated Completion Date. Please be aware of the following:"
  
  Tradeoff list displayed in the dialog:
  • "The Completion Date will no longer auto-calculate based on Start Date, Production Hours, and Changeover Time."
  • "All downstream orders in this feedmill line will have their Start Date/Time and Completion Date recalculated based on your new date."
  • "If you later edit Start Date, Start Time, or Volume, the Completion Date will NOT auto-recalculate — it will retain your overridden value until you remove the override."
  
  Input field: User must type "Confirm" (case-insensitive).
  Buttons: [Proceed] (disabled until "Confirm" is typed) [Cancel]

- Upon confirmation, date and time pickers open allowing the user to set a custom Completion Date and time.
- The lock icon changes to an "unlocked" state.

**After Override:**
- The overridden Completion Date is stored and used as the basis for the next order's Start Date/Time in the scheduling cascade.
- All downstream orders in the same feedmill line recalculate accordingly.
- The Completion Date remains fixed at the user's value and does NOT auto-recalculate when Start Date, Start Time, Volume, or Production Hours change — UNTIL the user removes the override.

**Removing the override:**
- When the Completion Date is in overridden/unlocked state, clicking the unlock icon shows a confirmation: "Revert to auto-calculated Completion Date?"
  Buttons: [Revert] [Cancel]
- Upon revert, the Completion Date recalculates from Start Date/Time + Production Hours + Changeover Time, the lock icon returns to locked state, and downstream orders recascade.

**Display after override:**
- Line 1: [Overridden Date] MMMM DD, YYYY
- Line 2: "Time: HH:MM AM/PM | ✎ Overridden" (smaller, grey text, with a pencil icon or "Overridden" label to indicate manual value)

**Editability by status:**
- Plotted, Cut, Combine, Hold: Override mechanism available
- In production, On-going *, Done, Cancel PO: Read-only (no override)

## 3. END DATE — EDITABLE ONLY WHEN DONE

Update the End Date editability rules:

- Done: Override mechanism available (auto-populated with current date/time when status changes to Done; user can override via lock icon + "Confirm" dialog as currently defined)
- All other statuses (Plotted, In production, On-going *, Cut, Combine, Hold, Cancel PO): Read-only

This replaces the previous rule where End Date was directly editable during "In production" and "On-going *" statuses. End Date is now ONLY editable (via override) when the order status is "Done".

## 7. DECIMAL FORMATTING RULES (GLOBAL)

Apply the following decimal formatting rules globally across the application:

**No decimal places (whole numbers):**
- Volume (MT) — both Suggested Volume and Original Volume (e.g., "20 MT", "19 MT")
- Number of Batches (e.g., "5")
- Number of Bags (e.g., "400")

**Always 2 decimal places:**
- Production Hours (e.g., "2.00 hrs")
- Changeover Time (e.g., "0.17")
- Run Rate (e.g., "21.23")

This applies across all views: interactive table, export, analytics, tooltips, modals, and any other display.

## 9. DRAG AND DROP — ROW-LEVEL DRAGGING

Update the drag-and-drop implementation so that users can initiate a drag from ANYWHERE on the row, not just the first column or a grip handle icon.

**Implementation:**
1. Make the entire row a draggable target. When the user clicks and holds anywhere on a row (except on interactive elements), the drag operation begins.

2. Keep the drag handle icon (grip dots/lines) visible on the left side of each row (before the Prio column) as a VISUAL INDICATOR of draggability, but do NOT restrict the drag initiation to only that icon.

3. On drag start:
   - Apply a slight elevation/shadow effect to the dragged row as a visual preview.
   - Optionally reduce opacity of the original row position to indicate displacement.

4. Exclude interactive elements from triggering drag:
   - Dropdowns (Status, HA Prep)
   - Text input fields (FG1, HA Available, SCADA, PV, FPR Notes, Special Remarks)
   - Calendar/time pickers (Start Date, Start Time)
   - Buttons (History button, lock icons)
   - Any clickable links or popovers
   
   These elements should handle their own click/focus events normally without initiating a drag.

5. On drop:
   - Place the row at its new position.
   - Re-sequence Prio numbers automatically.
   - Trigger the scheduling cascade to recalculate Start Date/Time and Completion Date for all affected orders within the same feedmill line.

6. Combined orders: When the lead order row is dragged, all combined orders in the group move together as a block. Individual non-lead combined orders cannot be dragged separately.

7. Recently Completed section rows (top 3 Done orders) are NOT draggable.

8. Cursor: Show `grab` cursor on hover over draggable row areas (excluding interactive elements). Show `grabbing` cursor while dragging.

9. Persist reordered positions in browser local storage so they survive page refreshes within the session.
