```
## FIX: NEXT 10 DAYS — TARGET DATE AND STATUS LOGIC

The target date calculation and status assignment is incorrect. Fix it based on the correct logic.

---

### CORRECT LOGIC — HOW IT WORKS

The system adds up volumes cumulatively: Due for loading + daily values, day by day. The moment the cumulative total MEETS or EXCEEDS the Inventory, that's where the "red zone" starts. The order should be produced BEFORE that red zone.

---

### STATUS DETERMINATION

**Stock Sufficient — No production needed:**
- If Due for loading ALONE is already >= Inventory → Stock Sufficient
  - Example: Due for loading = 25.2, Inventory = 23.4 → 25.2 >= 23.4 → Stock Sufficient
- If Bal. to produce is negative or blank → Stock Sufficient
- The product already has enough stock from day one

**Needs Production — Production required:**
- If Due for loading is BELOW Inventory → Needs Production
- A target date must be calculated
- The order should be produced BEFORE the date when cumulative loading meets/exceeds inventory

**Needs Production (Extended) — Production needed but won't complete in 10 days:**
- If cumulative loading never reaches Inventory within the 10-day window → special case
- Status = "Needs Production"
- Target date = last date in the range
- Add a note: "⚠ Inventory target unlikely to be met within the next 10 days. Production should start as early as possible."

### TARGET DATE CALCULATION

```
function calculateTargetDate(product) {
  const dueForLoading = product.due_for_loading;
  const inventory = product.inventory;
  const dailyValues = product.daily_values; // [{date, value}, ...]
  
  // STOCK SUFFICIENT CHECK
  // If due for loading already meets or exceeds inventory
  if (dueForLoading >= inventory) {
    return { 
      targetDate: null, 
      status: 'stock_sufficient',
      note: null
    };
  }
  
  // NEEDS PRODUCTION — calculate target date
  let cumulative = dueForLoading;
  let previousDate = null;
  let previousCumulative = dueForLoading;
  
  for (const day of dailyValues) {
    // Add this day's value (treat blank/null/0 as 0)
    const dayValue = day.value || 0;
    
    if (dayValue === 0) {
      // No loading this day, skip but track as potential previous date
      previousDate = day.date;
      continue;
    }
    
    const newCumulative = cumulative + dayValue;
    
    if (newCumulative >= inventory) {
      // This day hits or exceeds inventory — RED ZONE starts here
      // Target date = the day BEFORE this (last non-red day)
      
      if (previousDate) {
        return { 
          targetDate: previousDate, 
          status: 'needs_production',
          note: null
        };
      } else {
        // Very first day already exceeds — target is before the first day
        const dayBefore = subtractOneDay(dailyValues[0].date);
        return { 
          targetDate: dayBefore, 
          status: 'needs_production',
          note: null
        };
      }
    }
    
    cumulative = newCumulative;
    previousDate = day.date;
  }
  
  // Never reached inventory within 10 days
  // Target = last date, but add warning note
  return { 
    targetDate: dailyValues[dailyValues.length - 1].date, 
    status: 'needs_production',
    note: 'Inventory target unlikely to be met within the next 10 days as of ' + 
          formatDate(today) + '. Production should start as early as possible.'
  };
}
