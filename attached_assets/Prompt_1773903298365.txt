```
## FIX: AUTO-SEQUENCE — CORRECT SEQUENCING WITH NEXT 10 DAYS TARGET DATES

The Auto-Sequence feature is not correctly using the Next 10 Days target dates. Orders with inferred target dates from stock level data are being placed AFTER actual dated orders instead of being interleaved chronologically.

---

### THE PROBLEM

Current auto-sequence output (WRONG):
```
Prio 1: Salto Conditioner    | Avail: Mar 22    | Start: Mar 22
Prio 2: Gallimax Pigeon      | Avail: prio rep   | Start: Mar 22
Prio 3: Elite XP Startex     | Avail: prio rep   | Start: Mar 22
Prio 4: Gall 21 Chicken      | Avail: safety stk | Start: Mar 22
Prio 5: Gallimax 3 Maint     | Avail: safety stk | Start: Mar 23  ← TARGET: Mar 19!
...
```

Gallimax 3 Maintenance has a target date of March 19 from the Next 10 Days file, but it's placed at Prio 5 starting March 23. This is WRONG — it should be the FIRST order because March 19 is the earliest date among all orders.

---

### THE CORRECT LOGIC

**Step 1: Assign an effective date to EVERY order**

Every order gets an "effective date" used for sequencing:

```
function getEffectiveDate(order) {
  // Priority 1: Actual avail date (from SAP)
  if (isDateValue(order.avail_date)) {
    return { date: parseDate(order.avail_date), source: 'avail_date' };
  }
  
  // Priority 2: Inferred target date (from Next 10 Days)
  if (order.inferred_target_date) {
    return { date: parseDate(order.inferred_target_date), source: 'target_date' };
  }
  
  // Priority 3: No date — gap filler
  if (order.stock_status === 'stock_sufficient') {
    return { date: null, source: 'stock_sufficient' };
  }
  
  // Priority 4: No date, no match — true gap filler
  return { date: null, source: 'none' };
}
```

**Step 2: Sort ALL orders by effective date**

```
function sortByEffectiveDate(orders) {
  return orders.sort((a, b) => {
    const dateA = getEffectiveDate(a);
    const dateB = getEffectiveDate(b);
    
    // Both have dates — sort chronologically
    if (dateA.date && dateB.date) {
      if (dateA.date < dateB.date) return -1;
      if (dateA.date > dateB.date) return 1;
      
      // Same date — actual avail date takes priority over target date
      if (dateA.source === 'avail_date' && dateB.source === 'target_date') return -1;
      if (dateA.source === 'target_date' && dateB.source === 'avail_date') return 1;
      
      return 0;
    }
    
    // Order with date comes before order without date
    if (dateA.date && !dateB.date) return -1;
    if (!dateA.date && dateB.date) return 1;
    
    // Both have no date
    // Stock sufficient goes AFTER no-match gap fillers
    if (dateA.source === 'stock_sufficient' && dateB.source !== 'stock_sufficient') return 1;
    if (dateA.source !== 'stock_sufficient' && dateB.source === 'stock_sufficient') return -1;
    
    return 0;
  });
}
```

**Step 3: The sorted list IS the proposed sequence**

There is no separate "dated orders first, then gap fillers" logic. ALL orders with any kind of date (actual avail OR inferred target) are treated equally and sorted chronologically together. Orders without any date go to the bottom.
```

```
### START DATE CALCULATION AFTER SEQUENCING

Once the sequence is determined, calculate start dates working backwards from each order's effective date:

**For orders with effective dates (actual or target):**
- The order must COMPLETE on or before its effective date.
- Start Date = Effective Date - Production Hours - Changeover Time
- If the previous order's completion pushes this order's start later than needed, cascade forward and flag conflicts.

**For orders without effective dates (gap fillers, stock sufficient):**
- Start Date cascades from the previous order's completion + changeover.
- No deadline constraint.

**Example with times:**
```
Prio 1: Gallimax 3 Maint (80 MT, ~4.18 hrs, Target: Mar 19)

```
### SLIPPING ORDERS BEFORE ACTUAL DATED ORDERS

**Critical rule:** Even if the first order in the sequence has an actual avail date (e.g., March 22), the auto-sequence CAN and SHOULD place other orders BEFORE it if those orders have earlier effective dates.

**Example:**
```
If the line has:
- Order A: Avail = March 22 (actual)
- Order B: Avail = prio replenish, Target = March 19

Auto-sequence places Order B FIRST because March 19 < March 22.
Order B starts first, completes by March 19.
Order A starts after, completes by March 22.

This is CORRECT — dated orders are NOT always at the top.
```

**Another example with tight scheduling:**
```
- Order A: Avail = March 20 (actual), Production = 8 hrs
- Order B: Avail = safety stocks, Target = March 19, Production = 4 hrs
- Order C: Avail = prio replenish, Target = March 18, Production = 2 hrs

Auto-sequence:
Prio 1: Order C — Target: Mar 18 (start ~Mar 18 early, complete by Mar 18)
Prio 2: Order B — Target: Mar 19 (start after Order C, complete by Mar 19)
Prio 3: Order A — Avail: Mar 20 (start after Order B, complete by Mar 20) ✅

All orders meet their deadlines.
