## FIX: AUTO-SEQUENCE — SORT BY EFFECTIVE DATE ONLY, NOT BY AVAIL TYPE

The auto-sequence is incorrectly grouping orders by avail date TYPE (actual dates first, then prio replenish, then safety stocks). This is WRONG.

The auto-sequence must sort ALL orders by their EFFECTIVE DATE chronologically, regardless of whether the date comes from an actual avail date or an inferred target date from Next 10 Days. The avail date label (prio replenish, safety stocks, actual date) is IRRELEVANT for sorting — only the effective date matters.

---

### THE BUG

Current behavior (WRONG):
```
Group 1: Orders with actual avail dates (sorted by date)
Group 2: Orders with "prio replenish" (no date-based sorting)
Group 3: Orders with "safety stocks" (no date-based sorting)
```

This means:
- Gallimax 3 Maintenance (safety stocks, Target: Mar 19) is at Prio 10
- Salto Conditioner (Avail: Mar 22) is at Prio 1
- Mar 19 should come BEFORE Mar 22, but it doesn't because the app sorts by type first

### THE FIX — ONE UNIFIED SORT BY EFFECTIVE DATE

**Remove ALL type-based grouping.** There is no "avail dates first" or "prio replenish second" or "safety stocks third." There is ONLY one sort: chronological by effective date.

```
function autoSequence(orders, next10DaysData) {
  
  // STEP 1: Assign effective date to every order
  const ordersWithDates = orders.map(order => {
    let effectiveDate = null;
    let source = 'none';
    let priority = 4; // default: lowest (no date)
    
    // Check for actual avail date
    if (isDateValue(order.avail_date)) {
      effectiveDate = parseDate(order.avail_date);
      source = 'avail_date';
      priority = 1; // has a date
    }
    // Check for inferred target date from Next 10 Days
    else if (next10DaysData) {
      const stockMatch = next10DaysData.find(
        s => s.material_code === order.material_code_fg
      );
      
      if (stockMatch && stockMatch.target_date) {
        effectiveDate = parseDate(stockMatch.target_date);
        source = 'target_date';
        priority = 1; // has a date (SAME priority as actual avail date)
      } else if (stockMatch && stockMatch.status === 'stock_sufficient') {
        effectiveDate = null;
        source = 'stock_sufficient';
        priority = 3; // stock sufficient = near bottom
      } else {
        effectiveDate = null;
        source = 'none';
        priority = 2; // no match, gap filler
      }
    }
    
    return { ...order, effectiveDate, dateSource: source, sortPriority: priority };
  });
  
  // STEP 2: Sort — ONE unified chronological sort
  ordersWithDates.sort((a, b) => {
    
    // Both have effective dates — sort chronologically
    if (a.effectiveDate && b.effectiveDate) {
      const dateCompare = a.effectiveDate - b.effectiveDate;
      if (dateCompare !== 0) return dateCompare;
      
      // Same effective date — keep original relative order
      return 0;
    }
    
    // One has a date, the other doesn't — dated comes first
    if (a.effectiveDate && !b.effectiveDate) return -1;
    if (!a.effectiveDate && b.effectiveDate) return 1;
    
    // Neither has a date — sort by priority
    // Gap fillers (priority 2) before stock sufficient (priority 3)
    return a.sortPriority - b.sortPriority;
  });
  
  return ordersWithDates;
}
```

**CRITICAL:** The sort does NOT check `order.avail_date` type (prio replenish vs safety stocks vs actual date). It ONLY checks the `effectiveDate`. An order with "safety stocks" and Target: Mar 19 sorts BEFORE an order with Avail: Mar 22 because Mar 19 < Mar 22. Period.

### WHAT THE AUTO-SEQUENCE IS REALLY DOING

Think of it this way: The auto-sequence answers the question:

**"In what order should I produce these orders so that everything is ready when it needs to be?"**

It does NOT answer: "Which orders have actual dates vs prio replenish vs safety stocks?"

The avail date label is just a label. What matters is WHEN the order needs to be ready — whether that date comes from SAP (actual avail date) or from the Next 10 Days stock level file (inferred target date).
