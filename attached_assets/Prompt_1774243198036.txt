## 1. AUTO-SEQUENCE PREVIEW — USE ADJUSTED/SUGGESTED MT, NOT ORIGINAL

The auto-sequence preview must show the ADJUSTED volume (suggested MT) that has been rounded up to be divisible by the batch size, NOT the original SAP volume.

### THE BUG

When orders have volumes that aren't divisible by their batch size, the app adjusts/suggests a new volume (rounds up to nearest divisible number). This adjusted volume is what's used in the main interactive table. But the auto-sequence preview is showing the ORIGINAL volume instead.

**Example:**
```
Original volume: 5 MT
Batch size: 4.00
5 ÷ 4 = 1.25 batches — NOT whole

Adjusted/suggested volume: 8 MT
8 ÷ 4 = 2 batches — whole ✅

Main table shows: 8 MT ✅
Auto-sequence preview shows: 5 MT ❌ ← BUG
```

Another example:
```
Original volume: 10 MT
Batch size: 4.00
10 ÷ 4 = 2.5 batches — NOT whole

Adjusted/suggested volume: 12 MT
12 ÷ 4 = 3 batches — whole ✅

Main table shows: 12 MT ✅
Auto-sequence preview shows: 10 MT ❌ ← BUG
```

### THE FIX

When building the auto-sequence preview table, use the SAME volume values that are displayed in the main interactive table — which is the adjusted/suggested MT.

```javascript
// When populating auto-sequence preview rows:
// Use the adjusted volume, NOT the original SAP volume

const volumeToDisplay = order.adjusted_volume || order.suggested_volume || order.volume;
// NOT: order.original_volume or order.sap_volume
```

**Also recalculate dependent fields based on adjusted volume:**

| Field | Calculation |
|---|---|
| Volume (MT) | Use adjusted/suggested volume |
| Number of Batches | Adjusted Volume ÷ Batch Size |
| Number of Bags | Adjusted Volume × bags_per_MT |
| Production Hours | Based on adjusted batches |
| HA Info | Same as Number of Batches (auto-populated) |

**All these fields in the auto-sequence preview must use the adjusted volume, not the original.**

## 2. AUTO-SEQUENCE PREVIEW — REFLECT COMBINED ORDERS FROM MAIN TABLE

If orders are already combined in the main interactive table, the auto-sequence preview must reflect that — showing the combined group together with Lead/Child badges.

### CORE RULE

The auto-sequence preview mirrors the combined state from the main table. It does NOT create new combines or break existing ones.

- If orders are combined in the main table → they appear combined in the preview.
- If orders are NOT combined in the main table → they appear as standalone in the preview.
- Combined groups move as a BLOCK in the auto-sequence — the lead order determines the group's position.

### COMBINED GROUP SEQUENCING

When the auto-sequence algorithm positions orders:

**Step 1:** Treat each combined group as a SINGLE unit.
- Use the LEAD order's effective date (avail date or stock target date) to determine the group's position.
- The entire block (lead + all children) moves together.

**Step 2:** Insert the block at the correct chronological position.
- The lead order determines WHERE the block goes.
- Children follow immediately below the lead.

**Example:**
```
Main table has:
- Lead: FPR 260309, 80 MT, safety stocks, Target: Mar 28 (combined group)
  - Child 1: FPR 260309, 8 MT, safety stocks
  - Child 2: FPR 260309, 12 MT, safety stocks
  - Child 3: FPR 260309, 20 MT, safety stocks
- Standalone: FPR 260309, 80 MT, prio replenish, Target: Mar 28

Auto-sequence uses Lead's effective date (Mar 28) to position the entire block.
```
