## FIX: FPR NOTES AND SPECIAL REMARKS — RESTORE EDITABILITY

The FPR Notes and Special Remarks columns are no longer editable. Users cannot click to add or edit notes. This must be fixed immediately.

---

### THE PROBLEM

The recent changes (click-to-expand popup, ellipsis truncation, column resizing) likely broke the edit functionality. The click handler is now opening the read-only expanded popup instead of entering edit mode, or the cell is no longer registering as editable.

---

### THE FIX — TWO DISTINCT INTERACTIONS

FPR Notes and Special Remarks need TWO click behaviors:

**1. Single click on truncated text (with ellipsis) → Opens read-only expanded popup**
**2. Double click on the cell → Enters edit mode**

OR alternatively (simpler):

**1. Single click on the cell → Enters edit mode directly (like before)**
**2. The expanded popup is triggered by a small expand icon instead of clicking the text**

### RECOMMENDED APPROACH — SINGLE CLICK TO EDIT, ICON TO EXPAND

**Default state (text visible, no truncation):**
```
┌──────────────────────────────────────────┐
│ This is a note about the order           │
└──────────────────────────────────────────┘
```
- Single click → enters edit mode (textarea appears).
- Normal behavior, same as before.

**Default state (text truncated with ellipsis):**
```
┌──────────────────────────────────────────┐
│ This is a long note about the or...  ⤢  │
└──────────────────────────────────────────┘
```
- Single click on the TEXT → enters edit mode (textarea appears).
- Click on the ⤢ expand icon → opens the read-only expanded popup.

**Default state (empty — no notes):**
```
┌──────────────────────────────────────────┐
│ Add note...                              │
└──────────────────────────────────────────┘
```
- Single click → enters edit mode (textarea appears).
- Placeholder "Add note..." in grey italic.
