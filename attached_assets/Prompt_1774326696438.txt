## AUTO-SEQUENCE — TIEBREAKER LOGIC FOR SAME TARGET DATE

When two or more non-dated orders have the SAME target date from the Next 10 Days file, the auto-sequence needs a tiebreaker to determine which order goes first. Use the percentage gap between Due for Loading and Inventory.

---

### PRIORITY RULES (IN ORDER)

**Rule 1: Target date (primary sort)**
- Earlier target date = higher priority
- This is unchanged from before

**Rule 2: Percentage gap (tiebreaker for same target date)**
- When two orders share the same target date, calculate the fulfillment percentage
- The order with the LOWER fulfillment percentage (more still needed) gets higher priority

**Rule 3: Volume (second tiebreaker)**
- If two orders have the same target date AND the same fulfillment percentage
- Higher volume order goes first
- But also consider production time impact on downstream orders

---

### PERCENTAGE GAP CALCULATION

```javascript
function calculateFulfillmentPercentage(product) {
  const dueForLoading = product.due_for_loading;
  const inventory = product.inventory;
  
  // Avoid division by zero
  if (inventory === 0 || inventory === null) {
    return 100; // treat as fully fulfilled, lowest priority
  }
  
  // Fulfillment % = how much of the inventory target is already met
  const fulfillmentPct = (dueForLoading / inventory) * 100;
  
  return fulfillmentPct;
}

// Lower fulfillment % = more urgent = higher priority
// Higher fulfillment % = less urgent = lower priority
```

### UPDATED SORT ALGORITHM

```javascript
function autoSequenceSort(orderA, orderB, next10DaysData) {
  const dateA = getEffectiveDate(orderA, next10DaysData);
  const dateB = getEffectiveDate(orderB, next10DaysData);
  
  // Rule 0: Stock sufficient always goes last
  if (dateA.source === 'stock_sufficient' && dateB.source !== 'stock_sufficient') return 1;
  if (dateA.source !== 'stock_sufficient' && dateB.source === 'stock_sufficient') return -1;
  
  // Rule 0b: Orders with no date/no match go after dated orders
  if (dateA.date && !dateB.date) return -1;
  if (!dateA.date && dateB.date) return 1;
  
  // Both have dates — compare
  if (dateA.date && dateB.date) {
    
    // Rule 1: Earlier target date first
    const dateCompare = dateA.date - dateB.date;
    if (dateCompare !== 0) return dateCompare;
    
    // Rule 2: Same date — lower fulfillment % first (more urgent)
    const stockA = getStockData(orderA, next10DaysData);
    const stockB = getStockData(orderB, next10DaysData);
    
    if (stockA && stockB) {
      const pctA = calculateFulfillmentPercentage(stockA);
      const pctB = calculateFulfillmentPercentage(stockB);
      
      if (pctA !== pctB) {
        return pctA - pctB; // lower % = more urgent = comes first
      }
    }
    
    // Rule 3: Same date, same % — higher volume first
    const volA = orderA.adjusted_volume || orderA.volume;
    const volB = orderB.adjusted_volume || orderB.volume;
    
    if (volA !== volB) {
      return volB - volA; // higher volume first
    }
    
    // All same — maintain original order
    return 0;
  }
  
  // Both have no dates — maintain original order
  return 0;
}
```
