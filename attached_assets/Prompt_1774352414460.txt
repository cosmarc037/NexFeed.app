```javascript
function findInsertionPosition(newOrder, existingOrders) {
  const newDate = parseDate(newOrder.avail_date);
  
  // If the new order has no date (prio replenish, safety stocks)
  // Insert at the bottom of active orders
  if (!newDate) {
    const lastActiveIndex = existingOrders.findLastIndex(
      o => o.status !== 'Done' && o.status !== 'Cancel PO'
    );
    return lastActiveIndex + 1;
  }
  
  // If the new order has a date, find the correct chronological position
  for (let i = 0; i < existingOrders.length; i++) {
    const order = existingOrders[i];
    const orderDate = parseDate(order.avail_date);
    
    // Skip non-dated orders (they sit below dated orders)
    if (!orderDate) continue;
    
    // Skip Done/Cancelled orders
    if (order.status === 'Done' || order.status === 'Cancel PO') continue;
    
    // Insert BEFORE the first order with a date AFTER the new order's date
    if (orderDate > newDate) {
      return i;
    }
  }
  
  // If no order has a later date, insert after the last dated order
  const lastDatedIndex = existingOrders.findLastIndex(
    o => parseDate(o.avail_date) !== null && o.status !== 'Done' && o.status !== 'Cancel PO'
  );
  return lastDatedIndex + 1;
}

**Fix — Risk assessment logic:**

```javascript
function assessDeadlineRisk(existingOrders, newOrder, insertPosition) {
  const newProductionHours = newOrder.production_time + newOrder.changeover;
  const riskyOrders = [];
  
  // For each order AFTER the insertion point
  for (let i = insertPosition; i < existingOrders.length; i++) {
    const order = existingOrders[i];
    const deadline = parseDate(order.avail_date) || parseDate(order.target_date);
    
    if (!deadline) continue; // no deadline to miss
    
    // Check if the order already has a completion date
    if (order.completion_date) {
      const currentCompletion = new Date(order.completion_date);
      const newCompletion = new Date(currentCompletion.getTime() + newProductionHours * 60 * 60 * 1000);
      
      // If new completion exceeds deadline
      if (newCompletion > deadline) {
        riskyOrders.push({
          order: order,
          deadline: deadline,
          currentCompletion: currentCompletion,
          newCompletion: newCompletion,
          delayHours: newProductionHours
        });
      }
    }
  }
  
  return riskyOrders;
}
```

**Risk should only be flagged when:**
- An order's projected completion date (after adding the new order's production time) exceeds its deadline/avail date
- The delay is significant enough to actually cause a miss

**Risk should NOT be flagged when:**
- Adding 34 minutes to an order that has days of buffer before its deadline
- Orders without deadlines
