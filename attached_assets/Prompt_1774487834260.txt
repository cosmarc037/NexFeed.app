## FIX: COMMENT TRIANGLE INDICATOR — NOT SHOWING

The triangle indicator is not appearing on cells where comments have been left. Fix this to ensure the triangle always shows on the specific cell that has comments.

---

### THE PROBLEM

1. User leaves a comment on a cell → no triangle appears
2. Triangle should show per CELL, not per row
3. Each cell independently tracks its own comments and shows its own triangle

---

### THE FIX

### STEP 1: EVERY CELL MUST BE position: relative

For the triangle to appear in the top-right corner, the cell container MUST have `position: relative`:

```css
/* Every cell in the order table */
td,
.table-cell,
.order-cell {
  position: relative;
  overflow: visible; /* triangle must not be clipped */
}
```

### STEP 3: RENDER TRIANGLE WHEN CELL HAS COMMENTS

```jsx
function TableCell({ orderId, columnName, children }) {
  const comments = getCommentsForCell(orderId, columnName);
  const hasComments = comments && comments.length > 0;
  
  return (
    <td 
      className="order-cell" 
      style={{ position: 'relative', overflow: 'visible' }}
      onContextMenu={(e) => handleRightClick(e, orderId, columnName)}
    >
      {/* Triangle indicator — ONLY on cells with comments */}
      {hasComments && (
        <div 
          className="cell-comment-indicator"
          title={`${comments.length} comment${comments.length > 1 ? 's' : ''}`}
          onClick={(e) => openCommentPopover(e, orderId, columnName)}
        />
      )}
      
      {/* Cell content */}
      {children}
    </td>
  );
}
```

### STEP 4: COMMENT DATA STRUCTURE — PER CELL

Comments are stored per cell, keyed by order ID + column name:

```javascript
// Data structure
const comments = {
  "order_123__volume_mt": [
    {
      id: "comment_1",
      user: "John",
      text: "Volume adjusted due to client request",
      timestamp: "2026-03-25T10:30:00",
      edited: false
    },
    {
      id: "comment_2",
      user: "Maria",
      text: "Confirmed with warehouse",
      timestamp: "2026-03-25T14:15:00",
      edited: false
    }
  ],
  "order_123__avail_date": [
    {
      id: "comment_3",
      user: "John",
      text: "Date changed per client request",
      timestamp: "2026-03-25T11:00:00",
      edited: false
    }
  ]
};

// Key format: {orderId}__{columnName}
function getCommentKey(orderId, columnName) {
  return `${orderId}__${columnName}`;
}

function getCommentsForCell(orderId, columnName) {
  const key = getCommentKey(orderId, columnName);
  return comments[key] || [];
}
```

### STEP 8: ADDING COMMENT → TRIANGLE APPEARS IMMEDIATELY

When user posts a new comment:

```javascript
function postComment(orderId, columnName, text) {
  const key = getCommentKey(orderId, columnName);
  
  if (!comments[key]) {
    comments[key] = [];
  }
  
  comments[key].push({
    id: generateId(),
    user: getCurrentUser(),
    text: text,
    timestamp: new Date().toISOString(),
    edited: false
  });
  
  // Trigger re-render of the cell so triangle appears
  updateCellState(orderId, columnName);
}
```

The triangle MUST appear immediately after posting — no page refresh needed.

### STEP 9: DELETING LAST COMMENT → TRIANGLE DISAPPEARS

When the last comment on a cell is deleted:

```javascript
function deleteComment(orderId, columnName, commentId) {
  const key = getCommentKey(orderId, columnName);
  comments[key] = comments[key].filter(c => c.id !== commentId);
  
  // If no comments left, remove the key
  if (comments[key].length === 0) {
    delete comments[key];
  }
  
  // Trigger re-render — triangle disappears if no comments left
  updateCellState(orderId, columnName);
}
```
