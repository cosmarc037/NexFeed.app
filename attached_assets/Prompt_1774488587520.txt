## FIX: COMMENTS — TRIANGLE ON WRONG CELL AND COMMENTS SHARED ACROSS CELLS

Two bugs:
1. Triangle indicator appears on the first column instead of the actual cell where the comment was left
2. Comments left on one cell (e.g., Material Code) show up when opening comments on other cells — comments are being shared across the entire row instead of being cell-specific

---

### ROOT CAUSE

The comment is likely being stored and keyed by ORDER ID only, not by ORDER ID + COLUMN NAME. This causes:
- All cells in the same row share the same comments
- The triangle renders on the first cell (or a default cell) instead of the specific cell

---

### FIX 1: COMMENT KEY MUST INCLUDE COLUMN NAME

```javascript
// WRONG — keyed by order ID only
const commentKey = `order_${orderId}`;

// CORRECT — keyed by order ID AND column name
const commentKey = `order_${orderId}__${columnName}`;
```

Every function that reads, writes, or checks comments must use the combined key:

```javascript
// Store comment
function postComment(orderId, columnName, text) {
  const key = `${orderId}__${columnName}`;
  
  if (!commentsStore[key]) {
    commentsStore[key] = [];
  }
  
  commentsStore[key].push({
    id: generateId(),
    user: getCurrentUser(),
    text: text,
    timestamp: new Date().toISOString(),
    edited: false
  });
  
  saveToStorage(commentsStore);
  triggerRerender();
}

// Get comments for a specific cell
function getCommentsForCell(orderId, columnName) {
  const key = `${orderId}__${columnName}`;
  return commentsStore[key] || [];
}

// Check if a specific cell has comments
function cellHasComments(orderId, columnName) {
  const key = `${orderId}__${columnName}`;
  return commentsStore[key] && commentsStore[key].length > 0;
}

// Delete comment from specific cell
function deleteComment(orderId, columnName, commentId) {
  const key = `${orderId}__${columnName}`;
  
  if (commentsStore[key]) {
    commentsStore[key] = commentsStore[key].filter(c => c.id !== commentId);
    
    if (commentsStore[key].length === 0) {
      delete commentsStore[key];
    }
  }
  
  saveToStorage(commentsStore);
  triggerRerender();
}
```

### FIX 3: COMMENT POPOVER — SHOW ONLY THAT CELL'S COMMENTS

When opening the comment popover, pass BOTH orderId and columnName:

```javascript
function openCommentPopover(event, orderId, columnName) {
  // Get comments for THIS SPECIFIC CELL only
  const cellComments = getCommentsForCell(orderId, columnName);
  
  setPopoverState({
    visible: true,
    x: event.clientX,
    y: event.clientY,
    orderId: orderId,
    columnName: columnName,  // MUST be included
    comments: cellComments
  });
}
```

The popover MUST use both orderId AND columnName when:
- Displaying comments → `getCommentsForCell(orderId, columnName)`
- Posting new comment → `postComment(orderId, columnName, text)`
- Editing comment → `editComment(orderId, columnName, commentId, newText)`
- Deleting comment → `deleteComment(orderId, columnName, commentId)`
