```javascript
function getAvailDateTooltip(order, n10dData) {
  const product = n10dData.find(
    p => String(p.material_code_fg) === String(order.material_code_fg)
  );
  
  if (!product) {
    return 'No stock level data available.';
  }
  
  const status = product.status;
  const completionDate = product.completionDate 
    ? formatDateFull(product.completionDate) 
    : '—';
  const availDate = product.availDate 
    ? formatDateFull(product.availDate) 
    : '—';
  const buffer = product.inventory > 0 
    ? (((product.inventory - product.due_for_loading) / product.inventory) * 100).toFixed(1)
    : '-100.0';
  
  let lines = [
    'Stock level data available.',
    `Completion: ${completionDate}`,
    `Avail: ${availDate}`,
    `Status: ${status}`,
    `Buffer: ${buffer}%`
  ];
  
  return lines.join('\n');
}
```

## 2. CURSOR — POINTER ON ICON HOVER

When the user hovers over the icon, show a pointer (hand) cursor so they know it's interactive.

### THE PROBLEM

The icon doesn't show a pointer cursor, making it hard for users to know they can hover/interact with it.

### THE FIX

```css
.avail-date-icon {
  cursor: pointer;
}
```

Also increase the hover target area without changing the visual icon size:

```css
.avail-date-icon {
  cursor: pointer;
  padding: 4px;
  margin: -4px;
  border-radius: 4px;
  display: inline-flex;
  align-items: center;
  justify-content: center;
}

.avail-date-icon:hover {
  background: rgba(0, 0, 0, 0.05);
}
```

| Property | Value |
|---|---|
| Cursor | Pointer (hand icon) |
| Padding | 4px (invisible extra hover area) |
| Margin | -4px (compensates for padding so layout doesn't shift) |
| Hover background | Very subtle grey (rgba(0,0,0,0.05)) |
| Icon size | Unchanged — keep as-is |
| Clickable area | Larger than visual icon — easier to hover |

### VISUAL — NO CHANGE TO ICON SIZE

The icon stays the same size visually. The padding trick just makes the invisible hover target bigger:

```
Visual:     [⚠]        ← same small icon
Hit area:  [  ⚠  ]     ← larger invisible hover zone
```
