## FIX: ORDER TABLE — TEXT SELECTABLE VS DRAG

Currently, clicking and dragging on an order row always initiates a drag/reorder action. This prevents users from selecting text in the row cells. Fix so that text is selectable, and drag only activates from a specific drag handle.

---

### THE PROBLEM

- User tries to select text in a cell (e.g., item description, material code)
- Mouse down + mouse move triggers row drag instead of text selection
- User cannot copy text from order rows

---

### THE FIX — USE DRAG HANDLE ONLY

Drag should ONLY initiate from the drag handle icon (⠿ / grip dots), NOT from anywhere on the row.

### STEP 1: ADD DRAG HANDLE COLUMN

Each row has a small drag handle icon on the left side. Only this icon triggers drag:

```jsx
function OrderRow({ order, onDragStart, onDragEnd, onDragOver }) {
  const [isDragging, setIsDragging] = useState(false);
  
  function handleDragHandleMouseDown(e) {
    // Only the handle triggers drag
    setIsDragging(true);
  }
  
  return (
    <tr
      draggable={isDragging}
      onDragStart={(e) => {
        if (!isDragging) {
          e.preventDefault();
          return;
        }
        onDragStart(e, order);
      }}
      onDragEnd={(e) => {
        setIsDragging(false);
        onDragEnd(e);
      }}
      onDragOver={onDragOver}
    >
      {/* Drag handle — ONLY this triggers drag */}
      <td className="drag-handle-cell">
        <div 
          className="drag-handle"
          onMouseDown={handleDragHandleMouseDown}
          onMouseUp={() => setIsDragging(false)}
        >
          ⠿
        </div>
      </td>
      
      {/* All other cells — text is selectable */}
      <td className="selectable-cell">{order.prio}</td>
      <td className="selectable-cell">{order.fpr}</td>
      <td className="selectable-cell">{order.material_code_fg}</td>
      <td className="selectable-cell">{order.item_description}</td>
      {/* ... other cells ... */}
    </tr>
  );
}
```

### STEP 2: PREVENT ROW-LEVEL DRAG

Remove any `draggable="true"` or `onDragStart` that is directly on the `<tr>` or on cell elements. Drag should ONLY come from the handle:

```javascript
// WRONG — do NOT put draggable on the entire row
<tr draggable="true" onDragStart={handleDrag}>

// CORRECT — row is only draggable when handle is engaged
<tr draggable={isDragging} onDragStart={handleDrag}>
```
