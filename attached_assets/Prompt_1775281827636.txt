## FIX: ACTIVE STATUS — ALWAYS ON MOST RECENT HISTORY ENTRY

The "Active" status badge should always be on the MOST RECENT history entry, regardless of whether it's a file upload, manual edit, or revert. The most recent entry is the current state of the Knowledge Base.

---

### THE PROBLEM

```
# | DATE                  | TYPE        | NOTES                              | STATUS
1 | Apr 4, 2026 1:30 PM  | Manual Edit | Manual edit — 1 row(s) modified    |          ← should be Active
2 | Apr 2, 2026 10:05 PM | File Upload | Uploaded: Knowledge Base.xlsx      | Active   ← WRONG
```

### THE CORRECT BEHAVIOR

```
# | DATE                  | TYPE        | NOTES                              | STATUS
1 | Apr 4, 2026 1:30 PM  | Manual Edit | Manual edit — 1 row(s) modified    | Active   ← CORRECT (most recent)
2 | Apr 2, 2026 10:05 PM | File Upload | Uploaded: Knowledge Base.xlsx      |          ← no longer active
```

### THE FIX

The "Active" badge is determined purely by position — the FIRST entry (index 0) in the history array (which is sorted newest first) is always the active one.

```javascript
function isActiveEntry(entry, history) {
  // Active = most recent entry = first in the list
  return history.length > 0 && history[0].id === entry.id;
}
```

### WHEN EACH ACTION HAPPENS

| Action | What happens to "Active" |
|---|---|
| Upload new file | New upload entry becomes #1 → Active |
| Manual edit saved | New edit entry becomes #1 → Active |
| Revert to older version | New revert entry becomes #1 → Active |
| Delete a history entry | If non-active deleted, no change. Active entry cannot be deleted |

### PROTECT ACTIVE ENTRY

The active (most recent) entry cannot be deleted or reverted (it's already active):

```javascript
function canDeleteEntry(entry, history) {
  // Cannot delete the active (most recent) entry
  return history[0]?.id !== entry.id;
}

function canRevertEntry(entry, history) {
  // Cannot revert to the active entry (already active)
  return history[0]?.id !== entry.id;
}
```

### AFTER REVERT — NEW ENTRY BECOMES ACTIVE

```javascript
function handleRevert(entry) {
  if (!entry.snapshot) {
    showNotification('error', 'Cannot revert — no snapshot available.');
    return;
  }
  
  // Replace KB data
  setKbData(JSON.parse(JSON.stringify(entry.snapshot)));
  
  // Create NEW history entry for the revert — this becomes the active entry
  const revertEntry = {
    id: Date.now(),
    type: 'revert',
    timestamp: new Date().toISOString(),
    summary: `Reverted to version from ${formatDateTimeFull(entry.timestamp)}`,
    details: `Knowledge Base reverted to: ${entry.summary}`,
    snapshot: JSON.parse(JSON.stringify(entry.snapshot)),
    user: getCurrentUser(),
  };
  
  // Add to top of history — becomes #1 (Active)
  addToHistory(revertEntry);
  
  showNotification('success', `Knowledge Base reverted successfully.`);
}
```
