## FIX: HIGHLIGHT DUPLICATE MATERIAL CODE ROWS IN RED

When the Knowledge Base contains rows with duplicate FG Material Code values, highlight those cells in light red so users can easily identify duplicates. This works similarly to the yellow highlight for edited cells — just a light red background, no border lines.

### THE PROBLEM

The Knowledge Base table can contain rows with duplicate FG Material Code values (either from the uploaded file itself or from manual edits). Currently, there is no visual indicator to help users spot these duplicates.

### THE FIX

#### DETECT DUPLICATE FG MATERIAL CODES

Compute a set of duplicate material codes by counting occurrences. A material code is considered duplicate if it appears more than once AND is not empty/blank:

```javascript
const duplicateMaterialCodes = useMemo(() => {
  if (!kbData || kbData.length === 0) return new Set();

  // Count occurrences of each FG Material Code
  const codeCount = {};
  kbData.forEach((row) => {
    const code = normalizeValue(row.fgMaterialCode);
    if (code === '' || code === '-') return; // Skip empty/placeholder values
    codeCount[code] = (codeCount[code] || 0) + 1;
  });

  // Collect codes that appear more than once
  const duplicates = new Set();
  Object.entries(codeCount).forEach(([code, count]) => {
    if (count > 1) {
      duplicates.add(code);
    }
  });

  return duplicates;
}, [kbData]);
```

#### HELPER TO CHECK IF A ROW HAS DUPLICATE MATERIAL CODE

```javascript
function isDuplicateMaterialCode(row) {
  const code = normalizeValue(row.fgMaterialCode);
  if (code === '' || code === '-') return false;
  return duplicateMaterialCodes.has(code);
}
```

#### RENDER TABLE CELLS WITH RED HIGHLIGHT FOR DUPLICATES

Apply light red background to the FG Material Code cell when its value is duplicated. The red highlight takes priority over the yellow edited highlight on the same cell. For other cells in the same row, they keep their normal or yellow highlight as applicable — only the FG Material Code cell itself gets the red highlight:

```jsx
{columns.map((col) => {
  const isEdited = isCellEdited(row._rowId, col.key);
  const isDuplicate = col.key === 'fgMaterialCode' && isDuplicateMaterialCode(row);

  // Determine which highlight class to apply
  // Red (duplicate) takes priority over yellow (edited) on the same cell
  let cellClass = 'kb-cell';
  if (isDuplicate) {
    cellClass += ' kb-cell-duplicate';
  } else if (isEdited) {
    cellClass += ' kb-cell-edited';
  }

  return (
    <td key={col.key} className={cellClass}>
      {isEditMode ? (
        <input
          type="text"
          value={row[col.key] ?? ''}
          onChange={(e) => handleCellChange(row._rowId, col.key, e.target.value)}
          className={`kb-cell-input ${
            isDuplicate
              ? 'kb-cell-input-duplicate'
              : isEdited
              ? 'kb-cell-input-edited'
              : ''
          }`}
        />
      ) : (
        <span>{row[col.key] ?? ''}</span>
      )}
    </td>
  );
})}
```

#### DUPLICATE DETECTION MUST UPDATE IN REAL-TIME DURING EDITS

When a user changes an FG Material Code value in edit mode, the duplicate detection should recalculate immediately since `duplicateMaterialCodes` is derived from `kbData` via `useMemo`, it will automatically recompute when `kbData` changes:

```javascript
function handleCellChange(rowId, field, newValue) {
  setKbData((prev) => {
    return prev.map((row) => {
      if (row._rowId === rowId) {
        const oldValue = row[field];
        if (oldValue !== newValue) {
          trackChange('modify', row, field, oldValue, newValue);
        }
        return { ...row, [field]: newValue };
      }
      return row;
    });
  });
  // duplicateMaterialCodes will auto-recalculate via useMemo dependency on kbData
}
```

This means:
- If a user edits a material code to match another existing row, both cells immediately turn red
- If a user fixes a duplicate by changing the code to a unique value, the red highlight disappears from both cells (assuming no other duplicates remain for that code)
- If a user edits a material code so that 3 rows now share the same code, all 3 cells turn red

#### DUPLICATE DETECTION AFTER FILE UPLOAD

Duplicates from the uploaded file itself should also be highlighted immediately after upload. Since `duplicateMaterialCodes` is computed from `kbData`, it will automatically detect duplicates in the uploaded data:

// No special handling needed — the useMemo on kbData handles it:

```

Same principle — since `duplicateMaterialCodes` derives from `kbData`, reverting or deleting a history entry that changes `kbData` will automatically recalculate duplicates:

```

```javascript
function getDuplicateCount(materialCode) {
  const code = normalizeValue(materialCode);
  if (code === '' || code === '-') return 0;
  return kbData.filter(
    (row) => normalizeValue(row.fgMaterialCode) === code
  ).length;
}
```
