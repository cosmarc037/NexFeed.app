## OBJECTIVE

When the user clicks `Refresh` for a specific line in the AI Auto-Sequence preview:

- only that line should show a loading state
- the line’s preview area should appear visually busy/in-progress
- other lines should remain unaffected and interactive

This is a per-line refresh indicator, not a full-screen blocking loader.

---

## 1. APPLY LOADING STATE PER LINE ONLY

Track refresh/loading state by line ID, not globally.

Example state:
```javascript
const [refreshingLines, setRefreshingLines] = useState({});
```

Or:
```javascript
const [refreshingLineIds, setRefreshingLineIds] = useState(new Set());
```

When refreshing Line 2:
- mark only `line_2` as loading
- do not block Line 1, Line 3, etc.

Example:
```javascript
setRefreshingLines(prev => ({
  ...prev,
  [lineId]: true,
}));
```

And clear it when the refresh finishes.

## 2. SHOW A LOADING OVERLAY ON THE CURRENT LINE’S CONTENT AREA

When a line is refreshing, visually overlay only that line’s main content area, including:
- AI strategy cards
- Before / After preview tables
- line-specific strategy preview section

Do not block:
- line tabs
- other line content
- global navigation

### Desired effect:
- line content looks slightly gray / opaque / disabled
- spinner/loading icon appears on top
- optional text:
  - `Refreshing AI strategies...`
  - or `Re-analyzing Line 2...`

Wrap the line-specific panel in a container like:

- but still allow the user to click:
  - other line tabs
  - other line views

- do not globally disable the page

### Important:

## 7. KEEP OTHER LINES FULLY USABLE

If Line 2 is refreshing:
- user can click Line 1, Line 3, etc.
- those lines should render normally
- those lines should not show any loading overlay
- they should remain fully interactive

- they should still see the loading overlay for Line 2

Once refresh completes:
- remove the overlay for Line 2
- show the refreshed strategy cards and preview

## 11. CLEAN STATE HANDLING

Recommended flow:
1. user clicks Refresh on active line
2. mark that line as refreshing
3. render overlay for that line panel
4. regenerate AI strategies only for that line
5. update line strategy state
6. clear refreshing state for that line
7. remove overlay

Make sure failed refresh also clears loading state.

Example:
```javascript
async function refreshLineStrategies(lineId) {
  try {
    setRefreshingLines(prev => ({ ...prev, [lineId]: true }));
    await regenerateStrategiesForLine(lineId);
  } finally {
    setRefreshingLines(prev => ({ ...prev, [lineId]: false }));
  }
}
```
