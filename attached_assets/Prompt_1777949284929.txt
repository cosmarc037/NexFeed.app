## 4. KEEP DONE LOGIC AS-IS, BUT MIRROR THE SAME LIMIT FOR CANCELLED

If done logic already caps visible done rows to 3, keep that unchanged.

Now implement similar logic for cancelled:

### Suggested helper:
```javascript
function getVisibleCancelledOrders(cancelledOrders) {
  return [...cancelledOrders]
    .sort((a, b) => {
      const dateA = new Date(a.cancelled_at || 0).getTime();
      const dateB = new Date(b.cancelled_at || 0).getTime();
      return dateB - dateA; // newest first
    })
    .slice(0, 3);
}
```

This ensures:
- only 3 most recent cancelled orders remain visible
- newest cancelled is first within the cancelled block
