## OBJECTIVE

Update the auto-sequence combination behavior so the combined total MT is calculated using the correct per-order basis:

1. If an order’s MT is app-adjusted (rounded up only because raw MT is not divisible by batch size), use the **raw volume** as the basis for combination
2. If an order’s MT is manually adjusted by the user (`volume_override`), use the **override volume** as the basis for combination
3. If an order’s MT is already divisible by batch size and therefore was not app-adjusted and not manually overridden, use the **current/base volume** as the basis for combination

After summing those correct per-order basis volumes, apply the normal batch divisibility adjustment to the final combined order.

Current behavior:
- combination sums only `total_volume_mt` (raw volume)
- child order `volume_override` is ignored
- if the main order has a stale `volume_override`, it can incorrectly carry into the combined order
- this can produce incorrect combined MT and incorrect production hours

This is not aligned with how users expect combining to behave.

## REQUIRED COMBINATION RULES

### Rule 1 — App-adjusted order
If an order was only rounded up by the app because its raw MT is not divisible by batch size:
- use the **raw volume** as the combination basis

### Rule 2 — User-adjusted / overridden order
If an order was manually overridden by the user:
- use the **override volume** as the combination basis

### Rule 3 — Already divisible / no adjustment case
If an order is already divisible by batch size and has no user override:
- use that normal volume as the combination basis

Then:
- sum the chosen basis volumes
- round the final combined total up to the nearest batch multiple
- unless the final sum is already divisible

## 1. ADD A HELPER TO DETERMINE COMBINATION BASIS PER ORDER

Create a helper that decides which volume to use for each order before combination.

### Suggested helper:
```javascript
function getCombinationBasisVolume(order) {
  const rawVolume = parseFloat(order.total_volume_mt ?? order.volume ?? 0) || 0;
  const batchSize = parseFloat(order.batch_size ?? order.batchSize ?? 0) || 0;
  const overrideVolume = parseFloat(order.volume_override);

  const hasUserOverride = !Number.isNaN(overrideVolume);
  if (hasUserOverride) {
    return {
      basisVolume: overrideVolume,
      basisType: 'user_override',
      rawVolume,
      batchSize,
    };
  }

  if (batchSize > 0) {
    const isDivisible = rawVolume % batchSize === 0;

    if (!isDivisible) {
      return {
        basisVolume: rawVolume,
        basisType: 'app_adjusted_use_raw',
        rawVolume,
        batchSize,
      };
    }
  }

  return {
    basisVolume: rawVolume,
    basisType: 'raw_divisible',
    rawVolume,
    batchSize,
  };
}
```

Important:
- app-adjusted orders should still use raw as input to the combination
- user override should override everything
- already-divisible orders just use their normal/raw value

## 2. CHANGE COMBINE SUMMATION TO USE COMBINATION BASIS, NOT JUST RAW TOTAL_VOLUME_MT

Current behavior likely looks like:
```javascript
const totalVolume =
  mainOrder.total_volume_mt +
  ordersToCombine.reduce((s, o) => s + (o.total_volume_mt || 0), 0);
```

Replace this with logic that uses `getCombinationBasisVolume(order)` for every order in the combine set.

### Suggested implementation:
```javascript
function calculateCombinedBasisVolume(orders) {
  return Number(
    orders.reduce((sum, order) => {
      const { basisVolume } = getCombinationBasisVolume(order);
      return sum + basisVolume;
    }, 0).toFixed(2)
  );
}
```

Then:
```javascript
const combineOrders = [mainOrder, ...ordersToCombine];
const combinedBasisVolume = calculateCombinedBasisVolume(combineOrders);
```

## 3. APPLY FINAL BATCH ADJUSTMENT ONLY AFTER THE BASIS SUM IS COMPUTED

After you compute the combined basis sum:
- apply the standard round-up logic to the final combined order total
- do not round each order individually before summing (except user overrides, which are taken literally)

### Suggested helper:
```javascript
function adjustVolumeToBatchCeiling(volume, batchSize) {
  const v = parseFloat(volume || 0) || 0;
  const b = parseFloat(batchSize || 0) || 0;

  if (b <= 0) return Number(v.toFixed(2));

  return Number((Math.ceil(v / b) * b).toFixed(2));
}
```

Then:
```javascript
const finalCombinedVolume = adjustVolumeToBatchCeiling(combinedBasisVolume, combinedBatchSize);
```

Important:
- `combinedBatchSize` should be the valid batch size for the combined order
- do not carry stale override logic into the final combined order
