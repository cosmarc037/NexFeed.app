Fix the grammar/pluralization for the order count labels in the Before/After headers so the app correctly shows “order” for 1 and “orders” for any other count.

## OBJECTIVE

The header currently shows incorrect grammar such as:
- `After — 1 orders`

This should instead be:
- `After — 1 order`

We need proper singular/plural handling for all count-based labels in these summary headers.

---

## REQUIRED RULE

Use:
- `1 order`
for exactly one

Use:
- `0 orders`
- `2 orders`
- `3 orders`
- etc.
for all other values

---

## 1. ADD A SHARED PLURALIZATION HELPER

Create a reusable helper for the word "order".

### Suggested helper:
```javascript
function formatOrderCount(count) {
  const n = Number(count || 0);
  return `${n} ${n === 1 ? 'order' : 'orders'}`;
}
```

---

## 2. USE THIS HELPER IN BEFORE/AFTER HEADER LABELS

Anywhere the app currently builds labels like:
- `Before — X orders`
- `After — X orders`

replace the hardcoded string with the helper.

### Example:
```javascript
const beforeLabel = `Before — ${formatOrderCount(beforeCount)}`;
const afterLabel = `After — ${formatOrderCount(afterCount)}`;
```
