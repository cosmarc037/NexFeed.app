## CONFIRMED BUSINESS LOGIC

The correct interpretation is:

- Inventory = current stock on hand
- Due for Loading (DFL) = demand / required dispatch quantity

### Interpretation rules
- If DFL > Inventory → not enough stock to meet demand → shortage / stockout risk
- If DFL < Inventory → enough stock to meet demand → buffer remains
- If DFL = Inventory → exactly enough stock

- Inventory = 10, DFL = 15 → short by 5
- Inventory = 10, DFL = 5 → buffer of 5

Create or confirm a shared helper like:

```javascript
function getFutureDispatchSupplyDemandContext(row) {
  const inventory = parseFloat(row.inventory ?? 0) || 0;        // stock
  const dueForLoading = parseFloat(row.due_for_loading ?? 0) || 0; // demand

  const balance = Number((inventory - dueForLoading).toFixed(2));

  return {
    inventory,
    dueForLoading,
    balance,
    isSufficient: balance >= 0,
    isShort: balance < 0,
    deficit: balance < 0 ? Math.abs(balance) : 0,
    buffer: balance >= 0 ? balance : 0,
  };
}
```

Interpretation:
- `balance >= 0` → inventory covers demand
- `balance < 0` → shortage

Use this helper consistently across the app.
