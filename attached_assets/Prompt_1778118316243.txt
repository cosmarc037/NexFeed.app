## REQUIRED RULE

When calculating totals for overview cards, line summaries, line header totals, or any aggregated MT display:

Use the final displayed order volume, which is the same value shown in bold in the table.

This applies regardless of whether the order volume is:
- raw and already divisible
- app-adjusted / rounded up
- user-overridden
- combined
- any other resolved final display state

Do NOT use:
- original/raw volume
- hidden pre-adjustment volume
- stale combined basis
- old source totals

## 1. DEFINE A SHARED "FINAL DISPLAYED VOLUME" HELPER

Create or reuse a helper whose sole purpose is to return the exact volume displayed in bold in the order table.

### Suggested helper:
```javascript
function getFinalDisplayedOrderVolume(order) {
  const candidates = [
    order.display_volume_mt,
    order._displayVolumeMT,
    order.final_volume_mt,
    order._finalVolumeMT,
    order.volume_override,
    order._combined_effective_volume,
    order._effectiveVolumeMT,
    order.total_volume_mt,
    order.volume_mt,
    order.volume,
  ];

  for (const val of candidates) {
    const parsed = parseFloat(val);
    if (!Number.isNaN(parsed)) {
      return Number(parsed.toFixed(2));
    }
  }

  return 0;
}
```

Important:
- the first priority should be the actual resolved display volume used by the row renderer
- if necessary, explicitly store `_displayVolumeMT` when building table rows
- the goal is for overview totals and row display to use the same source of truth

## 2. USE THIS HELPER FOR ALL OVERVIEW / LINE TOTALS

Anywhere the app calculates totals like:
- line total MT
- line overview card MT
- line header totals
- production load totals
- overview total assigned MT

replace raw summation with:

```javascript
function calculateDisplayedVolumeTotal(orders) {
  return Number(
    (orders || []).reduce((sum, order) => {
      return sum + getFinalDisplayedOrderVolume(order);
    }, 0).toFixed(2)
  );
}
```
