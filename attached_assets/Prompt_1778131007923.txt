## OBJECTIVE

We want the row-level sequence insights to already be available as soon as auto-sequence finishes.

That means:
- insights should be generated during the auto-sequence run itself
- by the time the preview is shown, the row insights should already be attached to each strategy/result

Additionally, we want to preserve the old/stable behavior:
- once the insights for that run are generated, keep them as-is
- do not regenerate row-level insights every time the user switches between Standard Sequence / AI Strategy 1 / AI Strategy 2

Strategy switching should be fast and should use already-generated insights stored in the result.

### During auto-sequence
Generate row-level insights for:
- Standard Sequence
- AI Strategy 1
- AI Strategy 2
(or whichever strategies are available for that line)

### After auto-sequence completes
The preview should already include:
- final ordered rows
- metrics
- row-level insights
- expanded insights if applicable

### While user switches strategies
- use the already-generated insights attached to that strategy
- do not regenerate just because the active tab/card changed

## 1. GENERATE INSIGHTS AS PART OF THE AUTO-SEQUENCE PIPELINE

Add row-level insight generation into the auto-sequence build flow itself.

For each line and each strategy result, the pipeline should produce:
- final ordered orders
- strategy metrics
- AI-generated row-level insights

### Example conceptual flow:
```javascript
for (const line of lines) {
  const standardResult = buildStandardSequence(line);
  const aiOption1Result = await buildAIStrategy1(line);
  const aiOption2Result = await buildAIStrategy2(line);

  const enrichedStandard = await generateRowInsightsForStrategy(standardResult);
  const enrichedAi1 = await generateRowInsightsForStrategy(aiOption1Result);
  const enrichedAi2 = await generateRowInsightsForStrategy(aiOption2Result);

  previewResults[line.id] = {
    standard: enrichedStandard,
    ai_option_1: enrichedAi1,
    ai_option_2: enrichedAi2,
  };
}
```

The important point:
- by the time preview is rendered, insights are already included

## 4. FREEZE INSIGHTS FOR THE CURRENT PREVIEW RUN

Treat the preview as a “frozen result package.”

Once auto-sequence completes:
- the row insights for that preview are fixed
- the app should not rewrite them during normal navigation inside that preview

This avoids:
- unnecessary latency
- drifting explanations
- inconsistent text after each switch

## 5. ONLY REGENERATE INSIGHTS WHEN THE ACTUAL SEQUENCE STATE CHANGES

Do regenerate insights if a new preview state is created, such as when:
- auto-sequence is run again
- user clicks refresh/re-analyze
- profitability changes the actual row ordering
- any action changes the final sequence content/positions

In those cases:
- build a new preview package
- generate fresh insights for that new sequence state

But not for simple viewing/switching.
