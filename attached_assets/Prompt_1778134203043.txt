## OBJECTIVE

The current Profit Score tooltip is too generic.

Current tooltip example:
- `Profit Score: 10.2`
- `Margin / (production hours + changeover)`

This only shows the formula conceptually, but it does not show the actual values used for that specific order.

We want the tooltip to show the row-specific values:
- actual margin
- actual production hours
- actual changeover
- actual formula using those values

This will make the Profit Score more transparent and understandable.

---

## REQUIRED BEHAVIOR

For each order row in AI-powered sequences, when hovering the Profit Score, the tooltip should show:

### Preferred format
```text
Profit Score: 10.2

Margin: 22.1%
Prod. Hours: 2.00
CO: 0.17

Profit Score:
22.1% / (2.00 + 0.17)
```

This is only an example — the actual numbers should come from that specific row.

## 1. TOOLTIP MUST USE ACTUAL ROW VALUES

For the hovered order, the tooltip should read the actual row-specific values:
- margin for that product/order
- production hours displayed for that row
- changeover displayed for that row
- profit score displayed for that row

Do not show only the generic formula label anymore.

---

## 2. USE THE SAME VALUES SHOWN IN THE TABLE

The tooltip must use the same values already shown elsewhere in the row:
- Margin = actual margin shown for that product
- Prod. Hours = actual row production hours
- CO = actual row changeover
- Profit Score = actual displayed score

This must match the visible row data exactly.

Do not use:
- raw/original hidden values
- default changeover if the row has a different CO
- stale margin source if the row already has a resolved value

## 3. TOOLTIP CONTENT FORMAT

Use a structure like this:

```text
Profit Score: {score}

Margin: {margin}%
Prod. Hours: {productionHours}
CO: {changeover}

Profit Score:
{margin}% / ({productionHours} + {changeover})
```

### Example
```text
Profit Score: 10.2

Margin: 22.1%
Prod. Hours: 2.00
CO: 0.17

Profit Score:
22.1% / (2.00 + 0.17)
```

Formatting notes:
- margin should show `%`
- production hours and changeover should be formatted consistently, e.g. `2.00`
- score should use the displayed row precision, e.g. `10.2`

## 4. ENSURE THE FORMULA USES THE ACTUAL VALUES FOR THAT ORDER

The formula shown in the tooltip should be based on:

```javascript
profitScore = margin / (productionHours + changeover)
```

Where:
- `margin` = actual margin for the order
- `productionHours` = actual row production hours
- `changeover` = actual row changeover

The tooltip should display those exact values in the formula preview.
