Improve how the app determines which AI strategy gets the "Recommended" badge, and update the default selection behavior so the recommended AI strategy is automatically selected when one exists.

---

## OBJECTIVE

Right now, the app determines the orange "Recommended" badge using an internal scoring system that includes metrics not currently visible in the strategy cards, such as:
- Min. Slack
- Violations / deadline violations

However, those fields are currently hidden from the UI, so the recommendation can feel opaque or hard to explain.

Also, even when an AI strategy gets the "Recommended" badge, the app still automatically selects the Standard Sequence by default. This is not the desired behavior.

We want to improve:
1. how recommendation is determined so it aligns better with what is actually visible and explainable
2. the default selection behavior so the recommended AI strategy is auto-selected when one exists

---

## CURRENT LOGIC FROM REPLIT

Current recommendation behavior has two stages:

### Stage 1 — Eligibility gates
A strategy is disqualified if:
- it has any deadline violations
- Min. Slack < 1.5 hours
- Faithfulness < 60
- Standard Sequence is always excluded by design

### Stage 2 — Weighted score
Among eligible AI strategies, score using:
- Min. Slack = 30%
- Risk level = 20%
- Faithfulness = 25%
- MTS adjusted = 15%
- Total margin / profitability = 10%

Then the higher-scoring AI strategy gets the "Recommended" badge.

## PROBLEM TO FIX

### 1. The recommendation may rely too much on hidden metrics
Since the UI currently hides:
- Min. Slack
- Violations

the recommendation can feel disconnected from what the user sees on the card.

If the recommendation is based heavily on hidden fields, it becomes harder to explain to clients and harder for users to trust.

### 2. Recommended strategy is not auto-selected
Even when an AI strategy has the orange "Recommended" badge, the app still auto-selects the Standard Sequence.

This is not the expected behavior.

Expected:
- if an AI strategy is recommended, it should be auto-selected
- Standard Sequence should only be auto-selected when no AI strategy is recommended

## REQUIRED CHANGES

### A. Improve recommendation logic
Recommendation should be determined in a way that is more aligned with the metrics shown in the card and easier to justify.

### B. Auto-select the recommended AI strategy

### C. Fall back to Standard only if no AI strategy is recommended

## 1. REWORK THE RECOMMENDATION LOGIC TO PRIORITIZE VISIBLE / EXPLAINABLE METRICS

Since Min. Slack and Violations are currently hidden, reduce reliance on hidden metrics for the visible recommendation badge.

- Faithfulness
- MTS adjusted
- Changeover time
- Time-saved
- possibly profitability / total margin if already available internally

## 2. USE HIDDEN METRICS ONLY AS SAFETY GATES, NOT AS THE MAIN WINNER-SCORING DRIVER

If Min. Slack and Violations are still important for safety, that is okay — but use them mainly as:
- disqualifiers / safety gates

Then once a strategy passes those gates, choose the winner primarily using visible and explainable metrics.

### Recommended structure

#### Eligibility / hard gates

- it has deadline violations
- faithfulness is too low

## 3. RECOMMENDATION SCORING SHOULD FAVOR METRICS THE USER CAN ACTUALLY SEE

After eligibility, score AI strategies primarily using:
- Faithfulness
- Changeover improvement / lower changeover
- Time-saved improvement
- MTS adjusted
- optionally profitability / margin if appropriate

### Suggested scoring emphasis
Example weighting direction:
- Faithfulness: high importance
- Changeover / time-saved: high importance
- MTS adjusted: medium importance
- Profitability: medium/low importance
- Hidden metrics: low or only gating

You do not have to use these exact weights, but the winner should make sense from what the card visibly shows.
