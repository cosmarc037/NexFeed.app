## 1. TREAT ACTUAL CHANGEOVER IMPROVEMENT AS A PRIMARY RECOMMENDATION FACTOR

If a strategy’s main objective is changeover reduction, then the strongest recommendation signal should be:

- lower actual changeover than the alternatives
- real negative time-saved value (actual savings)
- no invalid date/constraint violations

In this case:
- `4.33 hr` is clearly better than `5.17 hr`
- `-0.84 hr` is a real improvement

That should make this strategy highly competitive for `Recommended`.

---

## 2. DO NOT LET LOW FAITHFULNESS AUTOMATICALLY DISQUALIFY A GENUINELY BETTER RESULT

Faithfulness should still be considered, but a modest or low faithfulness score should not automatically block recommendation when:
- the measurable result is better
- the sequence is still valid
- the optimization objective was actually achieved in the metrics

Instead, low faithfulness should mean:
- the description or visible pattern may need refinement
not necessarily:
- the strategy is worse in practical terms

---

## 3. SEPARATE "QUALITY OF RESULT" FROM "QUALITY OF EXPLANATION ALIGNMENT"

Please treat these as two separate dimensions:

### Quality of result
- actual changeover reduction
- actual time saved
- valid date-safe sequence
- maybe profitability / MTS impact as secondary factors

### Explanation alignment / faithfulness
- how closely the final table visually reflects the AI’s intended narrative

A sequence can have:
- high practical value
- but only moderate narrative faithfulness

That should still allow it to be Recommended if the actual outcome is best.

---

## 4. IF FAITHFULNESS IS LOW, ADJUST THE DESCRIPTION RATHER THAN DISCARDING THE STRATEGY

If the AI strategy genuinely reduced changeover but the final visible clustering only partially matches the original wording, then:
- revise the wording
- soften over-specific claims
- keep the recommendation if the metrics justify it

For example, if the strategy title/description is too strong, update it so it better matches the actual sequence.

This is better than rejecting a useful strategy just because the explanation is imperfect.

---

## 5. USE FAITHFULNESS AS A MODERATOR, NOT A DOMINANT OVERRIDE

Recommended scoring should likely work like this:
- actual changeover / time-saved = primary driver
- validity / date safety = hard gate
- faithfulness = modifier / confidence signal
- MTS adjusted / profitability = secondary tiebreakers

That means faithfulness can lower confidence somewhat, but it should not completely override a clearly better actual result.

---

## 6. REVISIT THE MINIMUM FAITHFULNESS THRESHOLD FOR RECOMMENDATION

If the current logic blocks recommendation below 60%, that may be too rigid.

In this example:
- faithfulness = 55.5%
- but actual performance is clearly better

Please revisit whether:
- the threshold should be softened
- or faithfulness should be weighted, not hard-gated
for strategies that materially outperform the alternatives
