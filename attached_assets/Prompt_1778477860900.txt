## GROUPING RULE — USE AVAIL DATE

The per-day Daily utilization breakdown must be grouped by:

- **Avail Date**

This is the required grouping basis.

Do not group by:
- Start Date
- Estimated Completion Date
- Production Start Date
- any derived date unless it is explicitly the Avail Date bucket

If the selected sequence contains orders with different avail dates, every avail date represented in that strategy’s resulting sequence should appear in the expanded breakdown.

---

## HOW TO AGGREGATE EACH AVAIL DATE ROW

For each Avail Date in the selected sequence, aggregate:

1. **Orders**
   - total number of orders assigned to that Avail Date bucket

2. **Hours**
   - total used hours for that Avail Date bucket
   - this should include:
     - production hours
     - plus changeovers assigned to that date

3. **Utilization %**
   - `(used hours / 24) * 100`
   - because the plant operates 24/7

### Example
If Avail Date = May 14 and total used hours for that date = 18.0:
- utilization = `18.0 / 24 = 75.0%`
