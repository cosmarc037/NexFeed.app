Please implement a new feature for Powermix / Line 5 orders.

This is a new operational rule:
when certain qualifying orders are plotted on **Line 5**, the system must automatically create a corresponding additional linked order on a configured destination line using a configured percentage of the original order quantity.

This should be implemented as configurable logic, not hard-coded directly into the sequencing flow.

---

## 1. BUSINESS RULE

If an eligible FG is scheduled on **Line 5 (Powermix)**, a linked additional order must automatically be created using the configured:

- mapped SFG / material
- target line
- percentage of original volume
- batch value

This is not a move of the original order.
The original Line 5 order must remain.
The system should create an **additional linked order** on the configured destination line.

---

- Material = 3000000000248

- Batch = 4

because 200 × 85% = 170.

## 6. TRIGGER CONDITION

The auto-generation logic should trigger only when:
- the source order is assigned to **Line 5**
- and the source FG has an active Powermix Split Rule

If the order is not on Line 5, do nothing.

If there is no matching rule, do nothing.

---

## 7. GENERATED ORDER LOGIC

When a rule matches:

### Generated quantity
Use:
`generated_qty = original_qty * percentage / 100`

### Generated line
Use the configured target line.

### Generated material
Use the configured SFG/material code.

### Batch
Use the configured batch value.

### Quantity precision
For now, keep quantity to 2 decimal places unless the app already has a standard rounding rule.

---

## 8. TRACEABILITY / LINKAGE

The generated order must remain linked to the original Line 5 order.

Please store or track fields such as:
- parent/source order id
- source line
- auto_generated flag
- generation rule id
- generated_from_powermix flag

This is required so the app can:
- avoid duplicates
- update linked orders correctly
- cancel linked orders if source order is removed
- explain where generated orders came from

---

## 9. DUPLICATE PREVENTION

Do not create duplicate linked orders every time:
- the line is reloaded
- the sequence is re-run
- the preview refreshes

Before creating a generated order, check whether a linked generated order already exists for:
- the same source order
- the same rule

If it exists:
- update it if needed
- do not create a second duplicate row

This is very important.

---

## 10. UPDATE / DELETE BEHAVIOR

### If the original Line 5 order quantity changes:
Update the linked generated order quantity accordingly, unless there is already a future manual detach rule.

### If the original Line 5 order is removed:
Remove, cancel, or deactivate the linked generated order accordingly.

### If the original order changes to another line:
If it is no longer on Line 5, then the linked generated order should no longer remain active as a Powermix-generated order.

For now, simplest behavior is acceptable as long as source/child consistency is maintained.

---

## 11. UI LABELING / VISIBILITY

Generated linked orders should be identifiable in the UI.

Consider displaying something like:
- Auto-generated
- Powermix split
- Derived from Line 5 order

This helps users understand why the order appeared on another line.

---

## 12. FUTURE-PROOFING

Although the current data suggests these are the only products relevant to Line 5 today, the implementation must be configurable so users can later:
- change percentage
- change target line
- change batch
- add new FG rules
- deactivate old rules

Do not hard-code the current mapping permanently into business logic.

Use the configuration table as the source of truth.
