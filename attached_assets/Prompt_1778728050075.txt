## 1. MOVE THE SPLIT EXPLANATION OUT OF VOLUME COLUMN AND INTO FPR NOTES

Please remove the split explanation text from the Volume (MT) column for Powermix-generated orders.

The app should no longer show text like:
- `Original: 56.7 MT`
- `(81% of 70 MT)`

under the volume value for generated Powermix rows.

### Required behavior
Keep the Volume (MT) column clean and only show the final displayed MT there.

Instead, place the explanation in the **FPR Notes** column.

### Recommended note format
For generated orders created from Line 5, populate FPR Notes with a descriptive note such as:

**Created from Line 5 order, producing 81% of [product name] from 70 MT source volume.**

Use the actual:
- percentage
- product name
- exact quantity basis used in the split

### Example
If the generated row is for Salto Stag Developer and the exact quantity basis used was 70 MT, then the note can say:

**Created from Line 5 order, producing 81% of Salto Stag Developer from 70 MT source volume.**

You may slightly improve wording if needed, but the meaning must stay clear.

For generated Powermix rows:

## 3. BATCH SIZE LOOKUP MUST USE MATERIAL CODE AGAINST POWERMIX SPLIT TAB

Clarification:
The app should use the relevant **material code** as the lookup key so it can resolve the correct batch size from the **Powermix Split** tab.

The problem is not that material code is being used.
The problem is that the batch size still appears to be resolved from the wrong source (such as Master Data or another fallback), instead of from the matched Powermix Split rule.

### Required behavior
For Powermix-applicable orders:
- use the relevant material code to find the matching Powermix Split rule
- once matched, use the **Batch Size from Powermix Split**
- do not override it with Master Data batch size if a valid Powermix Split match exists

### Goal
Material code should be the lookup key.
Powermix Split should be the source of truth for batch size in this feature.

### Example
If the matched Powermix Split rule says:
- material code maps to batch size = 2

then the generated order should use:
- Batch Size = 2

even if Master Data for that material shows batch size = 1

This same logic should also apply to the relevant Line 5 Powermix context where batch size needs to align with the Powermix Split rule.

---

## 4. SPLIT VOLUME MUST USE THE EXACT QUANTITY BASIS ACTUALLY USED BY THE APP

Problem:
There are still cases where the generated split volume appears to be calculated from the raw source quantity instead of the exact quantity basis actually used by the app.

### Required behavior
The split calculation must use the **exact quantity basis actually chosen by the app for the split**.

That same exact quantity basis must also be used in the FPR Notes text.

### Important consistency rule
Do not allow this mismatch:
- split math uses one quantity
- FPR note references another quantity

If the app used 70 MT for the split, then:
- the split math must use 70 MT
- the FPR note must say `from 70 MT source volume`

If the app used 72 MT for the split, then:
- the split math must use 72 MT
- the FPR note must say `from 72 MT source volume`

The calculation basis and note basis must match exactly.

Please audit the current split calculation path because it appears the app may still be using the wrong source quantity in some cases.
