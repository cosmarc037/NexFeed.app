Please fix the linked-order clickability issue for Powermix/source orders across tab/view navigation.

There is currently a bug where the item description text is clickable in one feedmill view, but after navigating to another view such as Powermix, the clickable behavior disappears until the page is refreshed or the user switches tabs again.

This needs to be fixed so clickable linked-order navigation remains reliable without requiring refresh.

---

## ISSUE OBSERVED

Current behavior:
- user is in FM3
- linked order item description is clickable
- user navigates to Powermix
- the linked item description is no longer clickable
- user must refresh the page or switch tabs again to restore clickability

This is incorrect.

The clickable item description should remain functional across Orders navigation without needing refresh.

---

## REQUIRED BEHAVIOR

For any valid linked Powermix/source order:
- the item description name should remain clickable every time the row is rendered
- regardless of whether the user arrived there from:
  - All Feedmills
  - Feedmill 1
  - Feedmill 2
  - Feedmill 3
  - Powermix

### Important
The app should not require:
- page refresh
- tab toggle workaround
- leaving and re-entering the page

to restore clickability.
