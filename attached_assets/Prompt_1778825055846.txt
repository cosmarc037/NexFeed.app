Please add another status dependency rule for Powermix-generated orders to prevent them from being reverted to a not-yet-started state once the linked Line 5 source order has already progressed.

This is a reverse status lock to preserve process consistency between the Line 5 source order and the generated split order.

---

## REQUIRED RULE

If the linked **Line 5 source order** is already:
- `On-going`
- or `Done`

then the linked **generated order** must not be allowed to be set back to a not-yet-started status such as:
- `Normal`
- `Plotted`
- blank
- or any equivalent not-yet-started state

### In other words
Once the source Line 5 order is already being produced or completed, the generated order cannot be reverted to a pre-start state.

This should be blocked.

---

## PURPOSE

This is to prevent inconsistent production states such as:
- source Line 5 order already `On-going` or `Done`
- generated order reverted back to `Plotted` / `Normal`

If the source order is already being produced or completed, that means the generated order should remain in a started/completed state as well, or at least should not be moved backward to a pre-start status.

---

## VALIDATION BEHAVIOR

When the user attempts to change the generated order status to a not-yet-started state, the app should:

1. identify the linked Line 5 source order
2. check the source orderŌĆÖs status

   - `Normal`
   - `Plotted`
   - blank
   - or equivalent not-started statuses
4. show a clear validation message

---

Use a message similar to:

**Cannot revert this generated order to a not-yet-started status because its linked Line 5 source order is already On-going or Done.**
