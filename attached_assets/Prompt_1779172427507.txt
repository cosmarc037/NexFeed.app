Please revise the Demand Profile calculation and Detailed Demand View logic based on the clarified business requirement.

The current Detailed Demand View uses inventory-style logic, but the intended requirement is different.

---

## BUSINESS REQUIREMENT

For a selected demand period, we want to compare:

- the **historical demand benchmark**
against
- the **current fulfilled volume**
- the **current planned / nasa pipeline volume**
- and determine the **remaining gap**

Example:
- Historical reference = January 2024 demand
- Current period = January 2025 as of today
- Then show:
  - how much is already fulfilled
  - how much is already planned / nasa pipeline
  - how much is still lacking

This is not mainly an inventory coverage calculation.

---

## IMPORTANT CHANGE

Please do **not** use Future Dispatch / current inventory as the main comparison basis for this demand coverage logic.

Instead, use operational order volumes from the application.

### Use:
- **Done** orders = fulfilled volume
- pipeline statuses = planned / nasa pipeline volume
- **Cancel PO** orders must be excluded

---

## STATUS MAPPING

### Fulfilled
Count only orders with status:
- `Done`

### Planned / Nasa Pipeline
Count orders with these statuses:
- `Plotted`
- `Planned`
- `Cut`
- `On-going`
- `Combined with other PO`

### Excluded
Do **not** include orders with status:
- `Cancel PO`

Also make sure cancelled orders are not counted in either fulfilled or pipeline.

## CALCULATION LOGIC

### Historical Demand (MT)
Use the historical demand dataset as the benchmark.

Example:
- January 2024 demand for SKU X

### Fulfilled / Done (MT)
Sum of order volumes where status = `Done` for the current comparison period.

### Planned / In-Line / Pipeline (MT)
Sum of order volumes where status is one of:
- `Plotted`
- `Planned`
- `Cut`
- `On-going`
- `Combined with other PO`

### Excluded Orders
Do not count any orders with status:
- `Cancel PO`

### Remaining Gap (MT)
```text
Remaining Gap = Historical Demand - (Fulfilled + Planned / Pipeline)
```

If this becomes negative, it is acceptable to:
- show 0 remaining
or
- show an exceeded state if preferred

### Coverage %
```text
Coverage % = (Fulfilled + Planned / Pipeline) / Historical Demand * 100
```
