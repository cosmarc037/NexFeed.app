## 2. APPLY DEFAULT SORTING LOGIC

Please sort the Detailed Demand View table by default so that rows **nearest to covering the demand** appear at the top.

### Desired behavior
Rows with:
- higher coverage %
- lower remaining gap
- closer to the historical demand target

should appear first.

Rows with:
- large remaining gap
- very low coverage
- or 0% coverage

should appear lower in the table.

### Recommended default sort logic
Use:
1. **Coverage (%) descending**
2. then **Gap (MT) ascending**

This should prioritize SKUs that are closest to the demand target.
