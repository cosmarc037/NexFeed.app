Please refine the **Detailed Demand View** table UI in the Demand Profile page with the following improvements.

---

## 1. USE FUNNEL ICON FILTERS IN TABLE HEADERS

Please replace the current inline filter input fields under the table headers with a **funnel icon filter behavior**, similar to the filter interaction already used in other tables in the app (such as the Orders / Feedmills table).

### Required behavior
- each applicable column header should show a **funnel/filter icon**
- clicking the funnel icon should open a filter panel / dropdown / popover
- the filter UI should behave consistently with the existing filter pattern already used elsewhere in the application
- do not keep the always-visible text input fields directly under the headers

### Goal
Make the Demand table filtering experience visually consistent with the rest of the app.

### Applicable columns
At minimum, support funnel-style filtering for:
- FG Code
- Item Description
- Period
- Status

If practical, also allow filtering/sorting for numeric columns in a sensible way:
- Demand (MT)
- Done (MT)
- Plotted (MT)
- Gap (MT)
- Coverage (%)

---

## 2. KEEP HEADER LABELS IN A SINGLE LINE FORMAT

Please adjust the table header labels so they do not wrap into two lines like:

```text

(MT)
```

Instead, I want them displayed in a cleaner single-line format like:

```text

```

### Apply this to all relevant columns
Use:
- FG Code
- Item Description
- Period
- Demand (MT)
- Done (MT)
- Plotted (MT)
- Gap (MT)
- Coverage (%)
- Status

Please ensure the header row styling/layout is adjusted so these labels stay visually clean and readable.

---

## 3. DEFAULT THE VIEW TO TODAY’S CURRENT PERIOD

Please make the Demand Profile page automatically default to the **current period** when the user opens the page.

### Expected behavior
When the user first opens Demand Profile:
- the app should already show the current relevant time period by default
- so the user can immediately see the demand status for the current month (or current date depending on time grain)

### Default behavior by time grain
- If the default time grain is **Monthly**, show the **current month**
  - example: if today is January 2025, default to **January 2025**
- If the user switches to **Daily**, show the **current date**
- If the user switches to **Quarterly**, show the **current quarter**
- If the user switches to **Yearly**, show the **current year**

### Important
The user must still be able to manually change:
- period
- year
- month
- time grain

But on first load, the page should already open in the current time context.

### Goal
At first glance, the user should immediately see what the demand fulfillment / gap looks like for the current period.
