## FORECASTING REQUIREMENT

The **Smart Demand (MT)** value should be derived by analyzing the historical demand pattern for the same SKU across all available years.

The logic should use historical demand records relevant to the SKU and selected period, and generate one practical forecast value.

At minimum, the forecast should consider:

- all available historical years for the SKU
- recurring month pattern when month-based records exist
- overall demand trend over time
- average demand behavior
- whether demand is increasing, decreasing, or stable

---

## PRACTICAL IMPLEMENTATION RULE

Do not overcomplicate this with a heavy external AI integration unless one already exists in the app.

If there is no existing forecasting service, implement a lightweight deterministic forecasting method that behaves like a smart AI estimate.

Use this priority:

### Preferred forecast logic
If historical demand exists for the **same SKU and same month across multiple years**:
- use those values as the primary forecast basis

Then:
- calculate a weighted forecast favoring more recent years
- apply a simple trend adjustment if a clear upward or downward pattern exists

### Fallback logic
If same-month multi-year history is incomplete:
- use all available historical demand for the SKU
- compute a weighted average with recency bias
- if too little data exists, fall back to:
  - current historical demand value for the row, or
  - average available demand values for that SKU

### Guardrail
Never return:
- null
- NaN
- undefined
- negative values

If insufficient data exists, return a safe numeric fallback.

## SUGGESTED FORECAST APPROACH

Implement a helper like:

```ts
calculateSmartDemand({
  sku,
  selectedMonth,
  selectedYear,
  historicalRecords
}): number
```

Suggested logic:

1. Filter historical records for the same SKU
2. Prefer records matching the same month from previous years
3. Sort records by year ascending
4. Apply recency-based weights, for example:
   - oldest year = lower weight
   - most recent year = highest weight
5. Compute weighted average
6. If enough data points exist, apply a small trend adjustment based on year-over-year movement
7. Clamp result to minimum 0
8. Format for display using the same numeric formatting as other MT columns

Example weighting idea:

- Year -5: 10%
- Year -4: 15%
- Year -3: 20%
- Year -2: 25%
- Year -1: 30%

If fewer years exist, normalize weights across available records.

Trend adjustment should be modest and should not create extreme forecast spikes.
