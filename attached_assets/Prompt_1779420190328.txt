### New routing behavior
- If the qualifying source order is in **Line 5**, generate the linked order in **Line 2**
- If the qualifying source order is in **Line 7**, generate the linked order in **Line 6**

- `1000000000041`
- `1000000000543`

- `1000000000565`

#### Source Line 7 → Target Line 6
- `1000000001121`

### Expected result
The system should no longer assume that all Powermix-linked generated orders come from the same source line.

Instead, it must:
- detect that the source order qualifies under Powermix split rules
- check which source line the order is currently scheduled in
- create the linked/generated order in the correct destination line based on that source line

## 2. UPDATE THE PLANNED ORDER COPY LOGIC FOR LINE 5 SOURCE ORDERS

Please update the auto-generated order logic for **Line 5** source orders.

For qualifying source orders in **Line 5**, the source row contains:
- `FG`
- `SFG`
- `SFG1`

When the linked/generated order is created from a **Line 5** source order, do **not** copy:
- `FG + SFG`

Instead, use:
- source `FG` → generated order `FG`
- source `SFG1` → generated order `SFG`

### New Line 5 mapping
- Generated Planned Order FG = **Source FG**
- Generated Planned Order SFG = **Source SFG1**

### Example
If the source order planned-order values are:
- FG = `5925179`
- SFG = `5881724`
- SFG1 = `5924991`

Then the generated order must use:
- FG = `5925179`
- SFG = `5924991`

### Expected result
For **Line 5 → Line 2** generated orders, the app should copy:
- **FG**
- **SFG1**

and should **not** use the source `SFG` value as the generated order’s `SFG`.
