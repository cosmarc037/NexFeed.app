## 1. UPDATE THE GENERATED ORDER COPYING RULE

Please revise the generated-order copy behavior so that generated orders no longer copy the source **FG** into the generated order’s **FG** field.

### New client rule
For generated orders:
- copy only the **SFG1 planned order value** from the source order
- copy only the **SFG1 material code** from the source order
- place both into the generated order’s **SFG** fields
- the generated order’s **FG** may be set to:
  - `-----`
  - or left blank

### Expected result
Generated orders should no longer use the source FG as the generated FG.

Instead, the generated order should primarily carry the **SFG1-based values** in the **SFG** columns.

---

## 2. PLANNED ORDER MAPPING FOR GENERATED ORDERS

Please update the planned-order mapping as follows:

### Source → Generated mapping
- Source **SFG1 Planned Order** → Generated **SFG Planned Order**
- Generated **FG Planned Order** → `-----` or blank

### Expected result
The generated order’s planned-order section should show:
- **FG = blank or `-----`**
- **SFG = source SFG1 planned order**

### Example
If the source order has:
- FG = `5925179`
- SFG = `5881724`
- SFG1 = `5924991`

then the generated order should use:
- FG = `-----` or blank
- SFG = `5924991`

---

## 3. MATERIAL CODE MAPPING FOR GENERATED ORDERS

Please update the material-code copy logic as follows:

### Source → Generated mapping
- Source **SFG1 Material Code** → Generated **SFG Material Code**
- Generated **FG Material Code** should not be copied from the source FG for this flow

### Expected result
The generated order should carry the **SFG1 material code** in its **SFG material code** field.

The generated order’s FG material code should remain blank or follow the current app behavior if that field is intentionally left empty for generated rows.

---

## 4. STOP COPYING SOURCE FG INTO GENERATED FG

Please remove the previous generated-order logic that copied:
- source **FG** into generated **FG**

for this flow.

### Expected result
The generated order should no longer show the source FG planned order value as its FG.

Instead:
- FG should be blank or `-----`
- SFG should carry the source SFG1-based planned order value
- SFG material code should carry the source SFG1 material code
