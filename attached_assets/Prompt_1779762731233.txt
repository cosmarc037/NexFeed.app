Please update the **changeover calculation logic** in the order table and related sequencing logic so that changeover is no longer fully cumulative/stackable across all triggered rules.

---

## 1. REPLACE THE FULLY CUMULATIVE CHANGEOVER MODEL

The application currently calculates changeover as:

- `Base changeover + all triggered additional changeover penalties`

This means multiple triggered rules stack together cumulatively.

Please update this logic.

### New intended model
Changeover should now be calculated as:

- **Cleaning**
- plus **Change Die**

It should **not** add every triggered rule cumulatively.

### Expected result
The app should no longer sum:
- base changeover
- plus all color/category/diameter penalties together

Instead, it should use the new selective two-part model described below.

---

## 2. DEFINE THE NEW CHANGEOVER FORMULA

Please use this formula:

### New formula
- If **no cleaning rule** and **no change die rule** applies:
  - use **Base changeover** only

- If **cleaning applies** but **no change die** applies:
  - use the **highest applicable cleaning rule only**

- If **change die applies** but **no cleaning** applies:
  - use **Change Die** only

- If **both cleaning and change die** apply:
  - use:
    - **highest applicable cleaning rule**
    - **plus**
    - **change die**

### Important
Do **not** add Base changeover on top of Cleaning or Change Die.

Base changeover is only used when there is no cleaning and no die change.

---

## 3. TREAT DIAMETER CHANGE AS “CHANGE DIE”

Please treat pellet diameter change as the **Change Die** component.

### Rule
If:
- `A.diameter ≠ B.diameter`
- and both diameters are set

then add the configured **Change Die** value for that feedmill.

### Example values from current rules
- FM1 = `1.5 hr`
- FM2 = `1.0 hr`
- FM3 = `1.0 hr`

### Expected result
Diameter change contributes only the die-change value, not a stack of other diameter-related amounts.

---

## 4. TREAT COLOR / CATEGORY TRANSITIONS AS “CLEANING”

Please group the following rules under the **Cleaning** component:

- Yellow ↔ Brown
- Red → Any
- Green → Any
- Any → Red/Green
- Category change

### Important
If multiple cleaning-related rules are triggered at the same time, do **not** add them together.

Instead:
- evaluate all triggered cleaning rules
- choose the **single highest cleaning value**
- use only that one cleaning value

### Expected result
Cleaning is based on the **maximum applicable cleaning rule**, not the sum of all cleaning rules.

---

## 5. USE ONLY THE HIGHEST CLEANING RULE WHEN MULTIPLE CLEANING RULES APPLY

Please explicitly update the logic so that among all triggered cleaning rules, only the rule with the **highest value** is used.

### Example
If a transition triggers:
- Yellow → Red = `0.5 hr`
- Category change = `0.33 hr`

then Cleaning should be:
- `0.5 hr`

not:
- `0.83 hr`

### Expected result
The cleaning component should always be:
- `max(all triggered cleaning rule values)`

---

## 6. BASE CHANGEOVER SHOULD ONLY APPLY WHEN NO CLEANING OR CHANGE DIE IS NEEDED

Please revise the role of **Base changeover**.

### New rule
Use the Base changeover only if:
- there is **no cleaning trigger**
- and there is **no diameter/change die trigger**

### Example
If two consecutive orders are identical in relevant changeover dimensions, then:
- use Base changeover (e.g. `0.17 hr`)

But if there is:
- cleaning,
- or change die,
- or both,

then do **not** add base on top.

### Expected result
Base changeover becomes the fallback/default only when there is no actual cleaning or die-change event.

---

## 7. EXAMPLES TO ENFORCE

### Example A — Color change only
FM1 Base = `0.17`

Orders:
- Order A: `Poultry · Yellow · 4.00mm`
- Order B: `Poultry · Red · 4.00mm`

Triggered cleaning:
- Yellow → Red = `0.5`

No diameter change.

### Expected changeover
- `0.5 hr`

### Not expected
- `0.17 + 0.5 = 0.67 hr`

---

### Example B — Red to Yellow + diameter change
FM1 Base = `0.17`

Orders:
- Order A: `Poultry · Red · 4.00mm`
- Order B: `Poultry · Yellow · 3.00mm`

Triggered cleaning:
- Red → Any = `1.0`

Triggered change die:
- 4.00mm → 3.00mm = `1.5`

### Expected changeover
- `1.0 + 1.5 = 2.5 hr`

### Not expected
- `0.17 + 1.0 + 1.5 = 2.67 hr`

---

### Example C — No meaningful change
FM1 Base = `0.17`

Orders:
- Order C: `Poultry · Yellow · 3.00mm`
- Order D: `Poultry · Yellow · 3.00mm`

No cleaning.
No diameter change.

### Expected changeover
- `0.17 hr`

---

### Example D — Category + color + diameter change
FM1 Base = `0.17`

Orders:
- Order A: `Swine · Yellow · 3.00mm`
- Order B: `Poultry · Red · 4.00mm`

Triggered cleaning rules:
- Yellow → Red = `0.5`
- Category change = `0.33`

Use highest cleaning only:
- `0.5`

Triggered change die:
- 3.00mm → 4.00mm = `1.5`

### Expected changeover
- `0.5 + 1.5 = 2.0 hr`

### Not expected
- `0.5 + 0.33 + 1.5`
- or `0.17 + 0.5 + 0.33 + 1.5`
