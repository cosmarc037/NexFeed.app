## 1. FIX RUN-RATE LOOKUP FOR GENERATED ORDERS DURING CROSS-LINE MOVEMENT

There appears to be an issue in the plant-level auto-sequence cross-line movement logic for **generated orders**.

Generated orders were previously updated so they can now be moved across lines during auto-sequencing, subject to:
- historical run-rate eligibility
- non-overlap checks
- normal line-scoring logic

However, generated orders still do not appear to move even in cases where:
- another partner/eligible line has lower queue time
- and master data already contains run rates for both lines

### Likely issue
The run-rate gate for generated orders may still be using the wrong material-code field when checking eligibility.

### Expected result
Generated orders should correctly pass the run-rate eligibility gate when their corresponding material has valid run rates in master data for the destination line.

---

## 2. FOR GENERATED ORDERS, USE SFG MATERIAL CODE AS THE LOOKUP KEY

Please update the cross-line movement eligibility logic so that for **generated orders**, the app uses the order’s:

- **SFG Material Code**

as the lookup key for run-rate eligibility.

### Important
This is specific to **generated orders**.

For normal orders, preserve the existing logic that uses:
- **FG Material Code**

### Expected result
The app should no longer rely on FG-based generated-order identity for cross-line run-rate evaluation.

---

## 3. MATCH THE GENERATED ORDER’S SFG MATERIAL CODE AGAINST FG MATERIAL CODE IN MASTER DATA

Please implement the following mapping rule for generated orders when checking master-data run-rate support:

### Generated-order run-rate lookup rule
- **Generated Order SFG Material Code**
  should be matched against
- **Master Data FG Material Code**

### Reason
In the current generated-order design, the generated order’s meaningful production identity is carried on the **SFG side**, while the master-data run-rate record appears to be stored under the **FG material code** field.

### Expected result
If a generated order has:
- `SFG Material Code = 3000000000248`

and master data has:
- `FG Material Code = 3000000000248`
- with valid run rates on `L1` and `L2`

then the generated order should be considered eligible for cross-line movement to Line 1 or Line 2, subject to the other existing rules.
