# 11. FULFILLMENT DETECTION RULE

If an order in **Demo All Feedmills** has a matching product/inventory in **Demo Future Dispatches** and the available inventory can satisfy that order’s required volume, then the app should detect that the order can be fulfilled from inventory.

### Example
Demo order:
- `Gallimax 200 MT`

Demo Future Dispatches inventory:
- `Gallimax 230 MT`

Since inventory can satisfy the order volume:
- the app should detect that this order is fulfillable from inventory

### Important
This should happen only within the **demo workspace** using the **demo Future Dispatches** dataset.

---

# 12. GENERATE A PENDING `PRE-ORDER` SUGGESTION BELOW THE ORDER TABLE

When the app detects that a demo order can be fulfilled from inventory, it should generate a suggested replenishment order below the order table.

### Expected behavior
The suggestion should:
- appear at the bottom of the sequence/order table
- be visually distinct (for example grayed out / pending style)
- show a badge such as:
  - `Pre-order`

### Important
This should be a **pending suggestion**, not an automatically committed order.

### Expected result
Users can review the suggested pre-order before deciding whether to include it.

---

# 13. ADD APPROVAL DECISIONS FOR THE PENDING `PRE-ORDER`

Please add decision controls on the generated pre-order suggestion, such as:

- `Approve`
- `Dismiss`

### Behavior
#### If user clicks `Approve`
- the generated pre-order should be added into the demo sequence
- this approval acts as the indicator that the original order was fulfilled using inventory
- the demo Future Dispatches adjusted inventory should update accordingly

#### If user clicks `Dismiss`
- nothing happens
- the pre-order is not added to the sequence
- the original order is not treated as fulfilled from inventory
- no inventory adjustment is applied from that suggestion

### Expected result
The user must explicitly confirm whether the inventory-based fulfillment should result in a replenishment/pre-order.

---

# 14. APPROVED `PRE-ORDER` SHOULD BECOME A NORMAL DEMO ORDER IN THE SEQUENCE

Once a user approves the generated pre-order suggestion, it should become a normal order within the **demo** sequence.

### Expected result
After approval, the pre-order should support normal demo order behaviors such as:
- sequencing
- drag/drop
- cut
- auto-sequence
- edits
- combination
- and other normal relevant order operations in Demo All Feedmills

---

# 15. DO NOT AUTOMATICALLY CHANGE THE ORIGINAL ORDER UNLESS THE PRE-ORDER IS APPROVED

Please preserve this rule:

- detection alone does not automatically finalize anything
- the order is only considered fulfilled from inventory once the user **Approves** the generated pre-order suggestion

### Expected result
If the user dismisses the suggestion:
- no fulfillment action is committed
- no adjusted inventory update should be applied from that suggestion
- no replenishment order is inserted

# 17. ADJUSTED INVENTORY FORMULA

Please calculate **Adjusted Inventory** as:

- `Adjusted Inventory = Inventory - satisfied order volume + Average Daily`

### Example
If:
- Inventory = `230 MT`
- Satisfied order volume = `200 MT`
- Average Daily = `20 MT`

Then:
- Adjusted Inventory = `50 MT`

### Important
This calculation should apply within the **demo Future Dispatches** dataset based on demo fulfillment approvals.

### Expected result
Adjusted Inventory reflects the demo inventory after approved fulfillment usage and replenishment logic.
