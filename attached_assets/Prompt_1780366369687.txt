Please add a **loading state to confirmation buttons** in the order-action modals so users get clear feedback that their click is already being processed.

---

## 1. ADD A LOADING STATE TO THE CONFIRM BUTTONS IN THESE MODALS

Please add a visible loading/processing state to the confirmation buttons in the following modals:

- `Cut Order`
- `Merge Back`
- `Uncombine Order`

### Goal
When the user clicks the confirm action, the UI should clearly show that the request is already being processed.

### Expected result
After clicking confirm, the button should no longer look idle/clickable with no feedback.

---

## 2. APPLY THE LOADING STATE TO THESE CONFIRM BUTTONS

Please apply the loading state to buttons such as:

- `Confirm Cut`
- `Confirm Merge`
- `Confirm Uncombine`

### Expected behavior
After the user clicks the confirm button:
- show a loading indicator/spinner
- disable repeated clicking
- optionally update the button text to a processing state

### Example text
- `Cutting...`
- `Merging...`
- `Uncombining...`

### Expected result
Users can clearly tell the action is already in progress.

---

## 3. PREVENT DOUBLE-CLICK / DUPLICATE SUBMISSIONS WHILE PROCESSING

Please disable the confirm button while the action is running.

### Goal
Avoid duplicate clicks or duplicate requests caused by lack of feedback.

### Expected result
Once the user clicks confirm, the button should not be clickable again until the action completes or fails.

---

## 4. KEEP THE LOADING STATE VISIBLE UNTIL THE ACTION COMPLETES OR FAILS

Please make sure the loading state remains active for the duration of the operation.

### Expected result
The loading state should:
- start immediately after confirm click
- remain visible while processing
- stop only when the action successfully finishes or an error occurs

---

## 5. IF THE ACTION FAILS, REMOVE THE LOADING STATE AND ALLOW RETRY

If the action does not complete successfully:
- stop the loading state
- re-enable the button
- allow the user to retry

### Expected result
The modal should not get stuck in a loading state after failure.

---

This request is only for improving the user feedback on confirm actions.
