# 1. USE STABLE BACKEND IDS / PRODUCTION-ENTITY IDS AS THE AI REFERENCE KEY

Please revise the auto-sequence flow so the AI sequences using stable backend identifiers instead of relying on row order, product names, or other unstable references.

### Important

- `order_id`
- `entity_id`
- `production_group_id`
- or other true backend-stable unique ID

If combining happens before AI sequencing, then the AI should rank the **post-combination production entities**, not only the raw individual orders.

### Expected result
Every AI sequencing recommendation is tied to a stable unique identifier that can be traced cleanly from:
- AI output
- preview table
- final apply-to-schedule
- logging/history
