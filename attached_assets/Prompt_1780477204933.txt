## 1. FIX THE SHUTDOWN DESTINATION LOGIC SO IT DOES NOT GET STUCK ON `LINE 1`

There is currently a problem where the Mash diversion prompt/modal keeps using **Line 1** as the shutdown destination, even when other lines are also in shutdown.

### Current issue
The row-level note and the Mash diversion modal continue to say:
- `Line 1 is shutdown`
- `Destination: Line 1`

even after another shutdown line is introduced.

### Likely issue
The flow appears to be using:
- a stale shutdown-line reference
- or only the first detected shutdown line
instead of reading the full current active shutdown-line set.

### Required behavior
The Mash diversion workflow must always reflect the **actual currently active shutdown lines**.

### Expected result
The feature should no longer default or get stuck on **Line 1** unless Line 1 is truly the only applicable shutdown destination.

## 2. SUPPORT MULTIPLE SHUTDOWN LINES AS DESTINATION OPTIONS FOR MASH DIVERSION

If more than one line is in shutdown, the Mash diversion modal should not behave as if there is only one fixed destination.

### Required behavior
If:
- only one shutdown line exists → it may remain auto-selected
- multiple shutdown lines exist → show all eligible shutdown lines as destination options

### Expected result
The user can choose among all currently valid shutdown-line destinations for the Mash order.
