## 1. FEED ALL RELEVANT SCHEDULING CONSIDERATIONS DIRECTLY INTO THE AI

Please make sure the AI receives all important sequencing context up front, so it can make the sequencing decision itself.

### This should include, at minimum:
- current lineup of orders
- future dispatch status / urgency status
  - `Critical`
  - `Urgent`
  - `Monitor`
  - `Sufficient`
- avail dates / deadlines / target dates
- changeovers
- material / category / color / diameter / form grouping context
- production time / run rate context
- current line and line constraints
- future dispatch pressure
- underloaded production day context
- line/day workload context
- any existing clustering / adjacency considerations
- generated / linked / combined entity relationships where applicable
- any other important sequencing considerations already known by the app

### Goal
The AI should have enough operational context to do the real sequencing itself.

### Expected result
The sequence should come from the AI’s actual reasoning, not from later correction.

## 2. MAKE SURE CRITICAL ORDERS ARE INCLUDED IN THE AI ANALYSIS

Please ensure that **Critical orders are always included in the AI sequencing input and output**.

### Important
The AI should not miss or omit Critical orders from its ranked response.

### Required behavior
If an order is marked as **Critical** in future dispatches, the AI must:
- recognize it as Critical
- include it in the ranked output
- and sequence it accordingly

### Goal
Critical orders should not be left out of the AI sequence.

### Expected result
No Critical order should disappear from the AI-ranked result.

## 3. USE FUTURE DISPATCH STATUS AS A CORE AI SIGNAL

Please make the AI explicitly use the order’s future-dispatch status as a core part of sequencing.

### Required interpretation
The AI should understand the business meaning of these statuses:

- **Critical**  
  Highest urgency; should be prioritized as immediate / today priority where applicable

- **Urgent**  
  Should be prioritized early

- **Monitor**  
  Can be advanced proactively when useful

- **Sufficient**  
  Has more scheduling flexibility

### Goal
The AI should know which orders are more urgent just from the future-dispatch status.

### Expected result
The sequencing reflects the actual urgency state of the orders.

## 4. HAVE THE AI RETURN A COMPLETE RANKED LIST FOR ALL INPUT ORDERS / ENTITIES

Please require the AI to return a ranked result for **every provided order/entity**.

### Important
If 20 orders/entities are sent into the AI, then 20 orders/entities must come back in the response.

### Goal
Avoid cases where omitted orders fall out of the analysis and then get placed incorrectly later.

### Expected result
The AI output is complete and covers the full sequence set.

## 6. MAKE SURE THE FINAL SORTING IS CORRECT

### Goal
The final result should look properly sorted and logically sequenced.

### Expected result

## 7. REDUCE DETERMINISTIC REFINEMENT TO LITTLE OR NONE

Please minimize deterministic refinement so it does not materially reshape the AI’s sequence.

### Important
The goal is:
- little to no deterministic refinement included in actual sequencing

### This means
Do not let rule-based post-processing take over the order after the AI has already ranked it, except only for the absolute minimum needed to validate output integrity if unavoidable.

### Goal
The final sequence should remain genuinely AI-driven.

### Expected result
The app should not be quietly re-sequencing the lineup after the AI responds.
