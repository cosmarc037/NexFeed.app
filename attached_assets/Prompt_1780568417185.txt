Please revisit the **AI vs Final Sequence ranking architecture**.

The current implementation still computes **Date Rank** and **Final Rank** in the frontend, even though the underlying **Suggested Date** and **Strategy Rank** are AI-generated. This does **not** fully align with the intended behavior.

If the goal is for the AI to truly own the sequencing logic, then **Date Rank** and **Final Rank** should also be **AI-generated / AI-owned**, not locally derived by the browser.

---

## 1. STOP TREATING `DATE RANK` AS A FRONTEND-COMPUTED VALUE

Right now, it appears that:

- **Suggested Date** is generated by the AI
- but **Date Rank** is still computed in the frontend by sorting those dates

### Concern
This means Date Rank is still a browser-side interpretation of AI output, not an AI-generated ranking dimension.

### Goal
If Date Rank is to exist, it should come from the AI itself rather than being computed locally.

### Expected result
Date Rank should no longer be a frontend-derived number.

---

## 2. STOP TREATING `FINAL RANK` AS A FRONTEND-COMPUTED VALUE

Right now, it appears that:

- **Final Rank** is still being computed by the frontend through sorting:
  - `suggestedDate ASC`
  - then `strategyRank ASC`

### Concern
This means the final visible ranking is still being assembled locally rather than returned by the AI.

### Goal
If the AI is meant to own sequencing, then Final Rank should also come directly from the AI.

### Expected result
Final Rank should no longer be browser-assembled from AI outputs.

---

## 3. MAKE THE AI RETURN `DATE RANK` IF THIS FIELD IS STILL NEEDED

If you want to keep **Date Rank** in the transparency table, then please have the AI explicitly return it.

### Goal
Ensure Date Rank reflects the AI’s own understanding of where the order sits in date-priority terms.

### Expected result
Date Rank becomes AI-generated rather than frontend-generated.

---

## 4. MAKE THE AI RETURN `FINAL RANK` AS THE ACTUAL FINAL ORDER

Please make the AI return **Final Rank** directly as the actual final sequence order.

### Important
The AI already has context on:
- suggested/effective date
- strategy
- urgency
- sequencing logic
- other ranking considerations

So the AI should be able to return the final intended order directly.

### Goal
Make the AI the true source of truth for final order placement.

### Expected result
Final Rank becomes AI-owned rather than mechanically derived in the frontend.

---

## 5. REDUCE OR REMOVE FRONTEND RECONSTRUCTION OF AI RANKING

Please reduce the need for the frontend to reconstruct rank positions from AI outputs.

### Concern
Right now, the browser is still interpreting:
- AI Suggested Date
- AI Strategy Rank

and then building:
- Date Rank
- Final Rank

This means the final sequence architecture is still partly split.

### Goal
Move to a cleaner model where the AI directly returns the ranking fields that the app displays.

### Expected result
The frontend should display AI-owned rank fields, not derive them.

---

## 6. ALIGN THE TRANSPARENCY TABLE WITH TRUE AI-OWNED RANKS

If the transparency table continues to show:
- Date Rank
- Strategy Rank
- Final Rank

then those ranks should all be truly AI-owned or AI-returned.

### Goal
Make the transparency table honestly represent what the AI actually decided.

### Expected result
